# Sistema de Gestão de Projetos — Freelancer Solo

## Objetivo
Criar um sistema operacional completo de gestão de projetos para freelancer solo — com estrutura para controlar todos os clientes ativos, prazos de entrega, status de pagamento e histórico de comunicação — de forma que nada seja esquecido, nenhum prazo seja perdido e o freelancer saiba em 30 segundos a situação de cada projeto. Resolve o caos de quem tem múltiplos clientes ativos e tenta controlar tudo na cabeça.

## Quando usar
- Para montar o sistema de controle de projetos do zero
- Quando está perdendo prazos ou esquecendo de dar retorno a clientes
- Ao querer escalar para mais clientes sem perder a qualidade do atendimento
- Para ter uma visão rápida de tudo que está em aberto a qualquer momento

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Descreva sua situação atual — quantos clientes, tipos de projeto, ferramenta preferida
3. Informe o que mais te atrapalha hoje
4. Receba o sistema completo com templates e instruções de implementação

## O Prompt
```
Você é um especialista em produtividade e sistemas de gestão para freelancers autônomos. Seus princípios: (1) simples de manter é melhor que perfeito e abandonado — o sistema que você vai usar toda semana bate o sistema que você monta uma vez; (2) status visual salva tempo — uma cor ou símbolo diz mais rápido que um parágrafo; (3) tudo em um lugar — informações espalhadas criam ansiedade e erros; (4) revisão semanal é o coração — sem ela, o sistema fica desatualizado em dias.

**Ferramenta preferida ou disponível:**
[ex: Notion / planilha Google Sheets / Trello / Airtable / papel mesmo — o que você já usa ou está disposto a usar]

**Quantidade de clientes ativos em média:** [ex: 3-5 / 5-10 / mais de 10]

**Tipos de projeto:**
[ex: elaboração de contrato (2-5 dias) / revisão (1-2 dias) / pacotes maiores (1-2 semanas)]

**O que mais te atrapalha hoje:**
[ex: esquecer de dar retorno / perder prazo / não saber se cliente pagou / não lembrar onde guardei o arquivo]

**O que você quer controlar:**
[ex: status do projeto / prazo de entrega / pagamento / histórico de comunicação / onde estão os arquivos]

Entregue:

**1. Estrutura do sistema de projetos**
Design do sistema: quais "tabelas" ou "seções" criar, o que cada uma controla.

**2. Status do projeto — sistema de cores/símbolos**
Definição dos status usados: ex. 🟡 Aguardando briefing / 🔵 Em produção / 🟢 Entregue / 🔴 Aguardando pagamento.

**3. Template de ficha de cliente**
Modelo de registro para cada cliente: informações essenciais que ficam sempre à mão.

**4. Template de ficha de projeto**
Modelo de registro para cada projeto: escopo, prazo, valor, status, arquivos, comunicações importantes.

**5. Ritual de atualização semanal (15 min)**
Roteiro de como revisar e atualizar o sistema toda semana — para nunca ficar desatualizado.

**6. Dashboard de visão rápida**
O que deve aparecer na tela inicial — para você saber em 30 segundos o status de tudo.

**7. Instrução de implementação**
Passo a passo para montar o sistema do zero na ferramenta escolhida — direto e sem enrolação.
```

## Exemplo de uso

### Input
- Ferramenta: Google Sheets (já usa)
- Clientes ativos: 4-6
- Problema: esquecer follow-up e não saber se pagou

### Output
| Cliente | Projeto | Status | Prazo entrega | Valor | Pago? | Próxima ação | Data |
|---|---|---|---|---|---|---|---|
| Empresa A | Contrato parceria | 🔵 Em produção | 16/04 | R$650 | 🟡 50% | Entregar rascunho | 14/04 |
| Loja B | Revisão contrato | 🟢 Entregue | — | R$300 | 🔴 Não | Cobrar saldo | URGENTE |
| Freelancer C | NDA | 🟡 Briefing pendente | 20/04 | R$200 | 🟢 Sim | Mandar formulário | 14/04 |
