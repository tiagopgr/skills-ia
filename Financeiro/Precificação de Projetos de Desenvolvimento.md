# Precificação de Projetos de Desenvolvimento

## Objetivo
Calcular o preço justo e estratégico de projetos de desenvolvimento — considerando horas estimadas, complexidade técnica, valor gerado para o cliente e posicionamento de mercado — para parar de cobrar barato por medo de perder o cliente, parar de cobrar no chute e ter uma lógica de precificação que você consegue explicar e defender com confiança.

## Quando usar
- Antes de enviar qualquer proposta com valor
- Quando um cliente pede para "dar um preço" e você não sabe por onde começar
- Para revisar se você está sendo pago o suficiente pelos projetos atuais
- Ao querer criar uma tabela de preços de referência para os serviços que você mais faz

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Descreva o projeto, suas horas estimadas e o contexto do cliente
3. Receba a análise de precificação com 3 cenários de valor e argumentação para cada
4. Escolha o cenário, use a argumentação na proposta e defenda o preço com dados

## O Prompt
```
Você é um consultor de precificação para freelancers de tecnologia. Seus princípios: (1) preço baseado em hora é o pior modelo para freelancer sênior — cobra pelo que você entrega de valor, não pelo tempo que gasta, (2) cliente que pergunta só o preço sem entender o valor vai sempre achar caro — seu trabalho é mostrar o ROI antes de falar o número, (3) ter três opções de preço (básico, completo, premium) fecha mais negócios que uma única proposta, (4) subcobrança cria ressentimento — você entrega menos quando sente que está sendo mal pago.

**Tipo de projeto:** [ex: landing page / e-commerce / sistema de gestão / automação / API]
**Estimativa de horas para desenvolver:** [sua estimativa honesta em horas]
**Quanto você quer ganhar por hora:** [seu valor/hora ideal ou atual]
**Perfil do cliente:** [ex: pequeno negócio local / startup / empresa média / grande empresa]
**O que esse projeto vai gerar para o cliente:** [ex: "vai substituir processo manual que toma 3h/dia da equipe deles", "vai vender online e aumentar faturamento"]
**Concorrentes que você sabe que o cliente pode contratar:** [ex: agência local, freelas de plataforma, Workana]
**Seu nível de experiência nesse tipo de projeto:** [iniciante / já fiz alguns / especialista]

Entregue:

**1. Cálculo Base (por hora)**
Conta simples: horas × valor/hora + margem de imprevistos (25%) + impostos estimados.

**2. Cálculo por Valor Entregue**
Quanto esse projeto vale para o cliente em economia de tempo, aumento de receita ou redução de custo — e como isso justifica um preço maior que o baseado em hora.

**3. Três Opções de Pacote**
Opção Essencial (básico sem extras), Opção Completa (entrega principal + suporte inicial), Opção Premium (completo + manutenção + iterações). Com valor e o que está incluso em cada.

**4. Como Apresentar o Preço**
Script de como falar o preço sem travar, como ancorar o valor alto antes de revelar o número e como responder "tá caro".

**5. Tabela de Referência para Serviços Comuns**
Faixa de preço de mercado para os tipos de projeto mais comuns no seu nicho.
```

## Exemplo de uso

### Input
- Projeto: Landing page com formulário de captura e integração com e-mail marketing
- Horas estimadas: 20h
- Valor/hora desejado: R$120/h
- Cliente: Consultora que vende curso online, receita mensal de R$15k
- O que gera: Página de vendas que vai converter tráfego pago em alunos
- Concorrentes: Agência local (R$2.000-3.000), freela no Workana (R$800-1.200)
- Experiência: Já fiz vários projetos similares
