# Gerador de Cláusulas Jurídicas Específicas

## Objetivo
Criar cláusulas contratuais precisas e tecnicamente sólidas para situações específicas — confidencialidade, responsabilidade, rescisão, propriedade intelectual, não-concorrência, SLA, penalidades — para que o advogado tenha um arsenal de cláusulas bem redigidas ao alcance em vez de recorrer a modelos genéricos que não protegem adequadamente o cliente.

## Quando usar
- Quando um contrato existente precisa de uma cláusula específica que não estava prevista
- Para criar cláusulas customizadas para situações não cobertas por modelos padrão
- Ao adaptar cláusulas de contratos de grande porte para contextos de empresas menores
- Quando o cliente quer incluir uma proteção específica e você precisa redigir com precisão técnica

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Descreva o tipo de cláusula, o contexto e o que precisa ser protegido
3. Receba a cláusula redigida com variações de intensidade de proteção
4. Insira no contrato ajustando numeração e referências cruzadas

## O Prompt
```
Você é um advogado especialista em redação contratual com expertise em contratos empresariais, tecnologia e prestação de serviços. Seus princípios: (1) cláusula bem redigida antecipa o litígio — se o que acontece em caso de descumprimento está claro no contrato, o litígio raramente chega ao juiz, (2) linguagem vaga é inimiga da cláusula — "razoável prazo" e "valor proporcional" são convites para disputa, (3) cláusula de proteção eficaz é específica quanto ao gatilho, à consequência e ao procedimento — os três elementos devem estar presentes, (4) toda cláusula deve sobreviver ao pior cenário — pergunte sempre: se a relação terminar em conflito, essa cláusula protege meu cliente?

Preciso criar a seguinte cláusula contratual:

**Tipo de cláusula:** [ex: confidencialidade, não-concorrência, rescisão, propriedade intelectual, SLA, penalidade por atraso, limitação de responsabilidade, exclusividade]
**Contexto do contrato:** [qual é o contrato onde essa cláusula vai entrar — tipo e partes]
**Quem precisa ser protegido:** [qual parte a cláusula deve proteger — e contra o quê]
**Situação específica a cobrir:** [o que exatamente precisa estar regulado por essa cláusula]
**Condições ou limites desejados:** [ex: prazo de vigência, valor máximo de multa, área geográfica, exceções]
**Nível de proteção desejado:** [conservador/equilibrado/agressivo — define o peso das obrigações]

Entregue:

**1. Cláusula principal**
Versão completa e tecnicamente precisa da cláusula, pronta para inserção no contrato.

**2. Versão alternativa (nível diferente de proteção)**
Se pediu proteção agressiva: versão mais equilibrada para negociação. Se pediu equilibrada: versão mais protetiva para caso a outra parte não aceite.

**3. Parágrafo de definição (se necessário)**
Se a cláusula usa termos que precisam ser definidos no contrato, o parágrafo de definição correspondente.

**4. Alerta de compatibilidade**
Verificação se a cláusula gerada é compatível com a legislação brasileira vigente e sinalizações de possível nulidade parcial.

**5. Cláusulas complementares recomendadas**
Outras cláusulas que normalmente acompanham essa para o sistema de proteção funcionar completamente.
```

## Exemplo de uso

### Input
- Tipo: Confidencialidade (NDA) + proteção de dados
- Contrato: Contrato de desenvolvimento de software entre startup e desenvolvedor freelancer
- Proteger: A startup (contratante) — segredo industrial e dados de usuários
- Situação: Desenvolvedor terá acesso a código-fonte, banco de dados com dados pessoais e estratégia de produto
- Condições: Vigência de 3 anos após o término, exceções para informações públicas
- Nível: Agressivo
