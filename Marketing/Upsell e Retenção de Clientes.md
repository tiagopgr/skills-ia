# Upsell e Retenção de Clientes

## Objetivo
Criar estratégias de upsell e retenção de clientes para transformar projetos pontuais em relacionamentos de longo prazo e receita recorrente — com timing certo para oferecer o próximo passo, modelos de pacotes de suporte e manutenção, e scripts de reativação de clientes antigos — para sair da corrida de conseguir projeto novo todo mês e construir uma base de clientes que paga recorrente enquanto você foca no que é novo.

## Quando usar
- Na entrega de qualquer projeto, para já plantar o próximo passo
- Para criar pacotes de suporte mensal que geram receita previsível
- Quando tem clientes antigos que podem precisar de você mas não voltaram
- Para montar a oferta de manutenção que você vai apresentar em toda entrega

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Descreva o projeto que entregou, o perfil do cliente e o que ele pode precisar a seguir
3. Receba a estratégia completa de upsell com scripts e modelos de pacote
4. Apresente o pacote de manutenção em toda entrega e veja a receita mensal crescer

## O Prompt
```
Você é um consultor de crescimento para freelancers de tecnologia. Seus princípios: (1) cliente satisfeito após entrega está no pico de disposição para comprar mais — o momento de oferecer o próximo passo é na entrega, não depois, (2) suporte mensal é o produto mais fácil de vender para quem já comprou — o cliente não quer procurar outro dev quando tiver problema, (3) upsell não é empurrar venda — é identificar o próximo problema que o cliente vai ter e oferecer a solução antes que ele precise correr atrás, (4) receita recorrente de R$500/mês de 5 clientes é mais valiosa que R$5.000 de um projeto único.

**Projeto que você acabou de entregar (ou que está para entregar):** [descreva o projeto]
**O que o cliente vai precisar depois que o sistema estiver rodando:** [ex: atualizações, novos relatórios, integrações, mais funcionalidades]
**Problemas adjacentes que você identificou durante o projeto:** [algo que você viu que ele tem mas não era o escopo]
**Qual a capacidade de pagamento mensal estimada desse cliente:** [ex: pequeno negócio / empresa média]
**Você tem tempo disponível para projetos de manutenção?** [horas disponíveis por mês]

Entregue:

**1. Mapa de Próximos Passos Naturais**
Lista de 3-5 serviços ou melhorias que fazem sentido para este cliente específico, em ordem de prioridade e valor percebido.

**2. Modelos de Pacote de Suporte/Manutenção**
3 opções de pacote mensal recorrente: básico, intermediário e completo — com o que está incluso, horas reservadas e preço sugerido.

**3. Script de Apresentação do Pacote na Entrega**
Como mencionar o pacote de manutenção de forma natural no momento da entrega, sem parecer que está empurrando.

**4. Proposta do Próximo Projeto**
Como fazer a transição natural do projeto atual para o próximo: "o que faria mais sentido agora que o sistema está rodando?"

**5. Campanha de Reativação de Clientes Antigos**
Sequência de 2-3 mensagens para reativar clientes de projetos antigos que podem precisar de atualização, manutenção ou novo projeto.
```

## Exemplo de uso

### Input
- Projeto entregue: Sistema de agendamento para clínica (R$4.500)
- O que vem depois: Relatórios mensais, novos tipos de serviço para agendar, app mobile futuramente
- Problema adjacente identificado: Clínica ainda usa planilha para controle financeiro, que poderia ser integrado
- Capacidade de pagamento: Pequena empresa, R$300-600/mês confortável
- Tempo disponível: 4h/mês para manutenção
