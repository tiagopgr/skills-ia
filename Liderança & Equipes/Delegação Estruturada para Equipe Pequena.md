# Delegação Estruturada para Equipe Pequena

## Objetivo
Estruturar o processo de delegação de tarefas e responsabilidades para equipe pequena — com briefing claro, critérios de sucesso definidos, prazo e forma de acompanhamento — para que o gestor delegue com confiança, o time execute sem precisar perguntar a todo momento e o resultado seja previsível sem microgestão.

## Quando usar
- Quando precisar delegar uma tarefa importante mas não sabe como briefar com clareza
- Quando algo delegado voltou errado e você quer criar um processo melhor
- Quando quiser aumentar a autonomia do time reduzindo as interrupções para você
- Quando um colaborador é novo e precisa de instrução mais detalhada
- Quando quiser padronizar como delega tarefas recorrentes

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Descreva a tarefa que quer delegar e para quem
3. Receba o briefing completo pronto para passar ao colaborador
4. Envie o briefing e defina o check-in

## O Prompt
```
Você é um especialista em gestão de equipes enxutas e delegação de alto desempenho. Seus princípios: (1) tarefa mal delegada é retrabalho garantido — clareza no briefing é economia de tempo, não burocracia, (2) quem delega sem critério de sucesso não pode reclamar de resultado abaixo do esperado — defina o que é "bom" antes, não depois, (3) autonomia se constrói — comece mais próximo e vá abrindo espaço conforme a confiança aumenta, (4) acompanhamento não é microgestão — é diferença de frequência e tom: check-in é suporte, não controle.

Crie o briefing de delegação para a seguinte tarefa:

**Tarefa a delegar:** [o que precisa ser feito — seja específico]
**Para quem:** [nome e nível de experiência do colaborador]
**Contexto:** [por que essa tarefa existe e qual é a importância]
**O que precisa ser entregue:** [o resultado concreto esperado — arquivo, resposta, processo, documento]
**Prazo:** [quando precisa estar pronto]
**Recursos disponíveis:** [acesso a sistemas, documentos, pessoas que podem ajudar]
**Restrições:** [o que não pode fazer ou alterar sem aprovação]
**Critério de qualidade:** [como você vai avaliar se foi bem feito]
**Nível de autonomia:** [pode decidir sozinho / consultar antes de agir / aprovar antes de enviar]
**Como prefere ser atualizado:** [status por WhatsApp / e-mail / reunião de check-in]

Entregue:

**1. Briefing completo para o colaborador**
Texto pronto para enviar ao colaborador — com contexto, tarefa, prazo, recursos, restrições e como pedir ajuda se travar.

**2. Critérios de sucesso explícitos**
Lista de o que significa "tarefa bem feita" — o colaborador deve conseguir autoavaliar antes de entregar.

**3. Perguntas que o colaborador provavelmente vai fazer**
3-5 dúvidas prováveis com respostas antecipadas — para evitar interrupções desnecessárias.

**4. Protocolo de check-in**
Quando e como você vai acompanhar — sem transformar em microgestão.

**5. O que fazer se o resultado não for o esperado**
Como dar o feedback de ajuste de forma que o colaborador aprenda sem se sentir atacado.
```

## Exemplo de uso

### Input
- Tarefa: Revisar 3 contratos de prestação de serviços recebidos de clientes esta semana
- Para quem: Ana, analista júnior — 3 meses na equipe
- Contexto: Clientes enviaram para análise antes de assinar, precisam de resposta em até 48h cada
- Entregáveis: Parecer em documento Word com pontos de atenção e sugestões de alteração
- Prazo: Segunda, quarta e sexta desta semana (um por dia)
- Recursos: Acesso ao Google Drive com modelos anteriores, checklist de revisão
- Restrições: Não enviar resposta para o cliente sem aprovação minha
- Critério: Usa o checklist de 12 pontos, identifica os 3 maiores riscos, sugere texto alternativo para cada
- Autonomia: Pode analisar e redigir, mas aprova comigo antes de enviar
- Updates: WhatsApp quando concluir cada análise

### Output
"Ana, preciso da sua ajuda com 3 revisões de contrato esta semana. Segue o briefing:

O quê: Revisar 3 contratos de prestação de serviços que recebemos de clientes. Cada cliente precisa de resposta nossa antes de assinar.

Entregas e prazos:
→ Contrato 1 (Cliente X): segunda até 17h
→ Contrato 2 (Cliente Y): quarta até 17h
→ Contrato 3 (Cliente Z): sexta até 17h

Como entregar: Documento Word com:
1. Resumo do contrato em 3-5 linhas (o que está sendo acordado)
2. Pontos identificados usando o checklist de revisão (Drive → Contratos → Checklist Padrão)
3. Para cada ponto de atenção: o texto atual + o que eu sugiro + por quê

Importante: Não envia nada para o cliente antes de eu aprovar. Me manda o Word pelo WhatsApp quando concluir cada um — vou dar feedback em no máximo 2h.

Se travar em algum ponto, me manda mensagem descrevendo onde está — não precisa esperar para perguntar."
