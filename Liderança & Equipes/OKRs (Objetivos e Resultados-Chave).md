# OKRs (Objetivos e Resultados-Chave)

## Objetivo
Criar OKRs bem definidos para a empresa, equipes e indivíduos — transformando metas vagas em objetivos inspiradores com resultados mensuráveis — para alinhar toda a organização em torno das prioridades.

## Quando usar
- No planejamento trimestral ou anual da empresa
- Ao definir metas de equipes e departamentos
- Para alinhar a equipe após mudança de estratégia
- Quando as metas existentes são vagas ou não inspiram ação

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Descreva as prioridades e o contexto do negócio
3. Receba OKRs estruturados e mensuráveis
4. Revise com a equipe e acompanhe semanalmente

## O Prompt
```
Você é um especialista em OKRs que sabe que o framework é simples mas a maioria erra. Os erros comuns: (1) confundir OKR com lista de tarefas, (2) criar Key Results não mensuráveis, (3) definir OKRs demais (foco é escolher poucos).

Crie OKRs para:

**Empresa:** [nome e o que faz]
**Período:** [Q1, Q2, anual]
**Situação atual:** [momento do negócio, desafios]
**Prioridades estratégicas:** [as 2-3 coisas mais importantes]
**Nível:** [empresa, departamento, indivíduo]
**Métricas atuais (baseline):** [faturamento, leads, churn, NPS]
**Meta ambiciosa:** [onde quer chegar]

Regras:
- Máximo 3-5 Objectives por nível
- Máximo 3-4 Key Results por Objective
- Objectives são qualitativos (sem números)
- Key Results são quantitativos (com números)
- Atingir 70% = sucesso

Entregue:
1. OKRs completos (Objective + Key Results + Dono + Iniciativas)
2. Conexão entre OKRs (cascata)
3. Sistema de acompanhamento
4. Scorecard do trimestre
5. Anti-patterns (o que NÃO fazer)
```

## Exemplo de uso

### Input
Empresa: SaaS de gestão de projetos (45 clientes, R$35K MRR)
Período: Q2 2025
Prioridades: Reduzir churn e aumentar receita por cliente
Métricas: MRR R$35K, Churn 6%/mês, NPS 42, ticket médio R$780

### Output
Objective 1: Tornar nosso produto indispensável para os clientes atuais

KR1: Reduzir churn mensal de 6% para 3% até fim de Q2
KR2: Aumentar NPS de 42 para 55+
KR3: Aumentar % de clientes que usam 3+ features de 35% para 60%
Dono: Head de Produto + Customer Success

Iniciativas:
- Implementar onboarding guiado para features subutilizadas
- Criar programa de health score com alerta automático

Check-in semanal:
1. "O KR está no caminho, em risco ou crítico?"
2. "O que aconteceu esta semana que moveu o KR?"
3. "O que precisa acontecer na próxima semana?"
