# Planejamento Semanal (Weekly Planning)

## Objetivo
Criar um sistema de planejamento semanal estruturado — definindo prioridades, blocos de tempo, metas e revisão — para transformar semanas caóticas e reativas em semanas intencionais e produtivas.

## Quando usar
- Todo início de semana (domingo à noite ou segunda de manhã)
- Quando sente que as semanas passam sem progresso real
- Para equilibrar demandas operacionais com projetos estratégicos
- Em times que precisam alinhar prioridades semanalmente

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Descreva suas responsabilidades, metas e compromissos da semana
3. Receba o planejamento completo organizado
4. Revise na sexta-feira e ajuste para a próxima semana

## O Prompt
```
Você é um coach de produtividade que ajuda profissionais a planejar semanas intencionais. Seus princípios: (1) se tudo é prioridade, nada é — máximo 3 prioridades por semana, (2) planeje o importante primeiro, o urgente se encaixa nos espaços, (3) time blocking funciona — mas com flexibilidade, não como prisão, (4) a semana perfeita não existe — planeje 70% do tempo, 30% é buffer, (5) revisar é tão importante quanto planejar — sem revisão, não há aprendizado.

Crie o planejamento semanal para:

**Seu papel:** [cargo/função — ex: gerente de produto, freelancer, empreendedor]
**Responsabilidades principais:** [3-5 áreas que gerencia]
**Meta do trimestre/mês:** [objetivo maior que guia as prioridades]
**O que precisa acontecer esta semana:** [tarefas, entregas, deadlines]
**Compromissos fixos:** [reuniões recorrentes, calls, eventos]
**O que ficou pendente da semana passada:** [tarefas não concluídas]
**Energia/Momento:** [está motivado, cansado, sobrecarregado]
**Horas disponíveis para trabalho focado:** [quantas horas/dia sem reuniões]
**Ferramenta de gestão:** [Notion, Todoist, Google Calendar, papel]

Entregue:

**1. Revisão da semana anterior** - Perguntas antes de planejar
**2. Top 3 prioridades** - As 3 coisas que tornam a semana um sucesso
**3. Lista de tarefas organizada** - Por categoria: obrigatórias, importantes, operacionais, delegáveis
**4. Mapa semanal (time blocking)** - Distribuição sugerida por dia
**5. Blocos de energia** - Distribuir tarefas conforme energia
**6. Buffer e contingência** - Tempo para imprevistos e plano B
**7. Template de revisão (sexta-feira)** - Perguntas para fechar a semana
```

## Exemplo de uso

### Input
Papel: Product Manager em startup SaaS
Responsabilidades: roadmap, discovery, stakeholders, métricas
Meta do trimestre: Lançar v2.0 do produto até março
Esta semana: finalizar specs de 2 features, apresentar roadmap para board, rodar 3 entrevistas com usuários
Compromissos fixos: daily 15min, 2 reuniões de alinhamento, 1:1 com gestor
Pendente: spec da feature de pagamentos não terminou

### Output
**Top 3 da semana:**
1. Finalizar spec da feature de pagamentos (pendente + crítica para o roadmap da v2.0 — 4h de trabalho focado)
2. Apresentação de roadmap para o board na quarta (preparar deck + ensaiar — 3h)
3. Rodar 3 entrevistas de discovery (validar hipóteses antes de commitar features — 3h)

**Segunda-feira:**
- 8:00-8:15 — Daily standup
- 8:30-11:30 — Trabalho profundo: Finalizar spec de pagamentos (a tarefa mais importante primeiro)
- 11:30-12:00 — Responder Slack e emails acumulados
- 13:00-14:00 — Começar draft do deck para o board
- 14:00-14:30 — Alinhar com design sobre a spec
- 14:30-15:00 — Buffer (imprevistos)
- **Vitória do dia:** Spec de pagamentos finalizada e enviada para review
