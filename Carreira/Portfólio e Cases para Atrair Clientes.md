# Portfólio e Cases para Atrair Clientes

## Objetivo
Estruturar o portfólio e os cases de projetos de desenvolvimento de uma forma que comunica valor para o cliente — não só "veja o que fiz" mas "veja o problema que resolvi e o resultado que entreguei" — para ter um portfólio que atrai o cliente certo, justifica um preço mais alto e convence sem precisar de um pitch de vendas longo.

## Quando usar
- Para criar ou reescrever o portfólio atual que só mostra prints de projetos
- Quando o cliente pede "você tem algum exemplo de trabalho?" e você não sabe o que mandar
- Para criar a página de portfólio no site ou no Notion que você vai linkar nas propostas
- Ao querer ter um asset que trabalha por você antes da primeira conversa com o cliente

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Descreva os projetos que você fez com o contexto do cliente e os resultados entregues
3. Receba os cases estruturados no formato certo e a arquitetura do portfólio completo
4. Publique no Notion, GitHub Pages ou site próprio e comece a linkar em propostas e no perfil

## O Prompt
```
Você é um consultor de marketing para desenvolvedores freelancers. Seus princípios: (1) portfólio que mostra código não converte cliente não-técnico — mostre o problema que existia, o que você construiu e o resultado que o cliente teve, (2) um case bem escrito vale por 10 projetos listados — profundidade vence quantidade, (3) o cliente olha o portfólio para responder uma pergunta: "esse cara já resolveu um problema parecido com o meu?" — estruture para responder isso, (4) prova social de qualquer tamanho funciona — depoimento de cliente pequeno é melhor que portfólio sem depoimento.

**Projetos que você quer transformar em case:** [descreva cada um: o que o cliente precisava, o que você construiu, tecnologias, resultado que o cliente teve]
**Seu nicho de atuação:** [ex: sistemas para pequenos negócios / landing pages / automações]
**Público-alvo do portfólio:** [quem vai ler — ex: donos de pequenos negócios / CTOs / gestores de marketing]
**Depoimentos de clientes que você tem (mesmo informais):** [o que clientes já disseram sobre o trabalho — mesmo por WhatsApp]
**Onde vai publicar o portfólio:** [ex: Notion / site próprio / LinkedIn / GitHub]
**Tom:** [ex: técnico e direto / acessível e narrativo / visual com pouco texto]

Entregue:

**1. Estrutura do Case por Projeto**
Template de case com as seções: contexto do cliente, problema que existia, o que foi construído (sem jargão), resultado mensurável e depoimento.

**2. Cases Escritos (prontos para publicar)**
Cada projeto que você descreveu transformado em case completo no formato definido acima.

**3. Arquitetura do Portfólio Completo**
Estrutura da página ou documento: seções, ordem, o que colocar em cada parte para guiar o visitante da curiosidade para o contato.

**4. Headline do Portfólio**
5 opções de frase de abertura do portfólio que capturam atenção e comunicam valor para o público-alvo.

**5. CTA do Portfólio**
Como terminar o portfólio com um chamado para ação que converte visitante em lead — sem ser agressivo.
```

## Exemplo de uso

### Input
- Projeto 1: Sistema de agendamento para clínica de fisioterapia — antes: agenda em papel + WhatsApp caótico. Construí: sistema web com login de paciente, calendário de agendamentos, confirmação por e-mail. Resultado: eliminou 3h de trabalho administrativo/dia
- Projeto 2: E-commerce para loja de roupas — antes: vendia só no Instagram com pedido por DM. Construí: loja WooCommerce com estoque, pagamento e frete. Resultado: 40 pedidos online na primeira semana
- Nicho: Pequenos negócios que querem sair de processos manuais
- Público-alvo: Donos de pequenos negócios
- Depoimento informal: "André, o sistema é incrível, não fico mais presa no telefone o dia todo" (Dra. Mariana)
- Onde publicar: Notion
- Tom: Acessível, narrativo
