# Conteúdo de Autoridade para Dev Freelancer

## Objetivo
Criar conteúdo de autoridade para desenvolvedor freelancer no Instagram ou LinkedIn — com ganchos que atraem o cliente certo, formatos que demonstram competência sem ser chato e ideias de conteúdo que funcionam mesmo sem grande audiência — para transformar suas redes sociais em canal de atração de clientes que te procura já convencido do seu valor, sem precisar vender no pitch.

## Quando usar
- Para criar o banco de ideias de conteúdo do mês em uma sessão
- Quando não sabe sobre o que postar ou só posta "terminei mais um projeto"
- Para criar conteúdo que gera DMs de potenciais clientes com interesse real
- Ao querer construir reputação no nicho sem precisar de milhares de seguidores

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Descreva sua especialidade, plataforma principal e o cliente que quer atrair
3. Receba o banco de ideias com roteiros prontos para produzir
4. Poste 2-3 vezes por semana e responda todos os comentários e DMs nas primeiras 2h

## O Prompt
```
Você é um estrategista de conteúdo especializado em profissionais de tecnologia que querem atrair clientes pelas redes sociais. Seus princípios: (1) dev que mostra o processo de trabalho atrai cliente que quer esse processo — bastidor converte mais que portfólio, (2) conteúdo para cliente não-técnico não pode ter jargão — "fiz um CRUD com autenticação JWT" não diz nada para o dono do restaurante, (3) o melhor post de um dev freelancer é "resolvi esse problema real de um cliente e foi assim" — case anônimo + solução + aprendizado, (4) consistência de nicho constrói autoridade — 30 posts sobre o mesmo tema valem mais que 30 posts sobre temas diferentes.

**Sua especialidade:** [ex: sistemas para pequenos negócios / automações / e-commerce / apps]
**Plataforma principal:** [ex: Instagram / LinkedIn / ambos]
**Perfil do cliente que quer atrair:** [ex: dono de pequeno negócio / gestor de marketing / CTO de startup]
**Seus maiores aprendizados como dev freelancer (pode ser pain):** [ex: aprendi que briefing ruim = retrabalho certo / aprendi a cobrar pelo valor, não pela hora]
**Projetos que você já fez que geraram resultado claro para o cliente:** [descreva brevemente]
**Quanto tempo tem para produzir conteúdo por semana:** [ex: 2h / 3h / 5h]

Entregue:

**1. Posicionamento de Conteúdo**
O tema central do seu conteúdo, o ângulo único que te diferencia de outros devs que postam, e a frase que resume o que seu conteúdo entrega.

**2. 15 Ideias de Post (banco para 1 mês)**
Para cada ideia: título/gancho, formato sugerido (texto corrido / carrossel / reel / imagem), público que vai resonar e estrutura resumida do conteúdo.

**3. 3 Roteiros de Post Completos**
Roteiro completo (gancho + desenvolvimento + CTA) de 3 posts de alto potencial para o seu perfil.

**4. Template de Post de Bastidor**
Estrutura para postar o processo de desenvolvimento de um projeto de forma que o cliente entende o valor sem entender o código.

**5. Como Transformar um Projeto em 3 Posts**
Como pegar um único projeto entregue e extrair 3 ângulos diferentes de conteúdo: o problema, a solução, o aprendizado.
```

## Exemplo de uso

### Input
- Especialidade: Sistemas web e automações para pequenos negócios
- Plataforma: Instagram + LinkedIn
- Cliente que quer atrair: Dono de pequeno negócio que usa processos manuais
- Aprendizados: "Brief mal feito me custou 20h extras", "cobrar por valor mudou tudo"
- Projeto com resultado: Sistema de agendamento para clínica eliminou 3h de trabalho manual/dia
- Tempo: 3h/semana para conteúdo
