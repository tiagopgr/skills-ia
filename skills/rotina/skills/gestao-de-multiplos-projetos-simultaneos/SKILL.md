# Gestão de Múltiplos Projetos Simultâneos

## Objetivo
Criar um sistema leve e prático de gestão para freelancers que tocam múltiplos projetos ao mesmo tempo — com painel de status, priorização por prazo e risco, rituais de acompanhamento e alertas de atenção — para nunca mais chegar na semana de entrega descobrindo que um projeto está atrasado, e ter clareza visual de onde cada projeto está a qualquer momento.

## Quando usar
- Quando você tem 2 ou mais projetos ativos ao mesmo tempo
- Para saber, em menos de 1 minuto, qual projeto precisa de atenção hoje
- Quando projetos começam a se misturar na cabeça e você perde contexto ao trocar
- Para evitar o pânico de "prometi entrega semana que vem e ainda falta muito"

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Liste seus projetos ativos com prazo, fase atual e o que falta fazer
3. Receba o sistema de gestão personalizado com painel, priorização e rituais
4. Monte no Notion, Trello ou planilha e atualize em 5 minutos por dia

## O Prompt
```
Você é um especialista em produtividade para desenvolvedores freelancers. Seus princípios: (1) sistema de gestão de projeto deve ser tão simples que você usa todo dia — sistema complexo vira mais um projeto a gerenciar, (2) o maior risco em múltiplos projetos não é o volume, é a troca de contexto — minimize o custo cognitivo de mudar de projeto, (3) cada projeto precisa de um status claro e uma única próxima ação definida, (4) deadline que não está visível todo dia é deadline que vai ser perdida.

**Projetos ativos agora:** [liste cada um com: nome do projeto / cliente / prazo de entrega / fase atual]
**Horas disponíveis por semana:** [total de horas de trabalho por semana]
**Ferramenta que você já usa ou quer usar:** [ex: Notion / Trello / planilha / papel / ClickUp]
**Maior problema na gestão atual:** [ex: "esqueço de coisas de um projeto enquanto estou no outro", "não sei o que fazer primeiro de manhã", "clientes ficam me perguntando status"]
**Você trabalha melhor em qual regime:** [ex: foco em 1 projeto por dia / alterna entre projetos no mesmo dia]

Entregue:

**1. Painel de Status dos Projetos**
Template de painel com: nome do projeto, cliente, prazo, fase atual, % concluído, próxima ação, bloqueios e data do último update. Formato de tabela adaptável para Notion/planilha.

**2. Sistema de Priorização Diária**
Como decidir em quais projetos focar cada dia: matriz de urgência + impacto adaptada para freelancer, com critérios de desempate.

**3. Rituais de Acompanhamento**
Check-in diário de 5 minutos (o que move hoje?) e revisão semanal de 20 minutos (onde cada projeto está vs. onde deveria estar?).

**4. Template de Contexto por Projeto**
Bloco de notas de projeto para você registrar onde parou, o que decidiu e o que está pendente — elimina o tempo de "onde eu estava mesmo?" ao retomar.

**5. Alertas de Risco**
Critérios que indicam que um projeto está em risco de atraso e o que fazer quando o alerta dispara.
```

## Exemplo de uso

### Input
- Projetos: (1) E-commerce "Moda X" / cliente Ana / prazo: 20 dias / fase: dev backend. (2) Landing page "Curso Y" / cliente Pedro / prazo: 7 dias / fase: ajustes finais. (3) API "Sistema Z" / cliente empresa / prazo: 45 dias / fase: planejamento
- Horas disponíveis: 40h/semana
- Ferramenta: Notion
- Problema: Chega de manhã sem saber o que fazer primeiro
- Regime: Prefere focar em 1 projeto por dia
