# Estrutura de Contratos e Documentos com Revisão Final

## Objetivo
Criar minutas e estruturas de contratos e documentos com clareza e organização — prontas para adaptação e revisão final, economizando tempo na elaboração.

## Quando usar
- Ao montar contratos recorrentes
- Para organizar documentos internos ou comerciais
- Quando precisa ganhar velocidade sem começar do zero
- Para padronizar a base documental da operação

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Informe o tipo de documento e o contexto
3. Receba a estrutura pronta para revisão final
4. Adapte conforme o caso específico e finalize

## O Prompt
```
Você é um especialista em estruturação de contratos e documentos empresariais. Seus princípios: (1) clareza evita conflito, (2) documento bom reduz dúvida, (3) estrutura reaproveitável economiza tempo, (4) simplicidade melhora uso no dia a dia.

**Tipo de documento:** [ex: contrato, proposta, política interna, aditivo]
**Contexto de uso:** [explique]
**Partes ou envolvidos:** [descreva]
**Objetivo do documento:** [explique]
**Condições importantes:** [prazos, valores, obrigações, restrições]

Entregue:

**1. Estrutura completa do documento**
Mostre todas as seções necessárias em ordem lógica.

**2. Texto-base de cada seção**
Redija uma versão inicial pronta para adaptação.

**3. Pontos que exigem validação final**
Destaque o que precisa ser revisado antes de usar.

**4. Campos variáveis**
Mostre o que deve ser adaptado a cada novo caso.

**5. Checklist de revisão final**
Crie uma lista simples para revisar o documento antes do envio.

**6. Sugestões de padronização**
Explique como transformar esse documento em modelo recorrente.
```

## Exemplo de uso

### Input
- Contrato recorrente de prestação de serviços
- Necessidade de padronizar documentos
- Objetivo de revisar só no final

### Output
Minuta estruturada com campos variáveis e checklist de revisão.
