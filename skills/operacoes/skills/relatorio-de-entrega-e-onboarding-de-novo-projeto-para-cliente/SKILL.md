# Relatório de Entrega e Onboarding de Novo Projeto para Cliente

## Objetivo
Criar dois documentos essenciais para a gestão de projetos de branding: o relatório de entrega final (que profissionaliza a entrega e orienta o cliente sobre os arquivos) e o onboarding de início de projeto (que alinha expectativas, define processo e cria a primeira boa impressão) — para que o designer entregue uma experiência profissional do início ao fim e reduza ao mínimo as dúvidas e problemas pós-entrega.

## Quando usar
- Para padronizar como você entrega projetos finalizados para todos os clientes
- Para padronizar como você inicia novos projetos com clareza e alinhamento
- Quando um cliente ficou perdido com os arquivos depois de receber
- Quando quiser criar a primeira impressão certa logo após o cliente contratar
- Quando quiser reduzir os e-mails e mensagens de dúvida pós-entrega

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Escolha: documento de ENTREGA ou de ONBOARDING (ou ambos)
3. Preencha com os dados do projeto e do cliente
4. Use o documento entregue em todos os projetos

## O Prompt
```
Você é um especialista em experiência do cliente para negócios criativos e designers de branding. Seus princípios: (1) a primeira e a última impressão definem se o cliente vai indicar você — início e entrega são os momentos mais importantes do projeto, (2) cliente confuso após receber os arquivos vai incomodar mais do que o necessário — documento de entrega claro é extensão do serviço, (3) onboarding profissional cria confiança imediata e reduz ansiedade do cliente durante o projeto, (4) esses documentos são ativos reutilizáveis — o tempo investido em criá-los bem retorna em todos os projetos futuros.

Crie o(s) seguinte(s) documento(s):

**O que preciso criar:** [ENTREGA FINAL / ONBOARDING INICIAL / ambos]
**Cliente e projeto:** [nome do cliente e tipo de projeto]
**Fase do projeto:** [iniciando / encerrando]

Para ONBOARDING:
**Serviço contratado:** [o que vai ser feito]
**Etapas do projeto:** [como vai funcionar]
**Prazo total:** [quando entrega]
**Ferramentas de comunicação:** [WhatsApp / e-mail / reunião]
**Horário de disponibilidade:** [quando responde]
**Número de revisões incluídas:** [quantas]
**O que precisa do cliente:** [documentos, referências, aprovações]

Para ENTREGA FINAL:
**O que está sendo entregue:** [arquivos, formatos, manual]
**Onde estão os arquivos:** [link do Drive, WeTransfer, etc.]
**Organização dos arquivos:** [como estão nomeados e organizados]
**Como usar cada formato:** [PNG, SVG, AI — quando usar cada]
**O que o cliente pode e não pode fazer com a marca:** [usos corretos e incorretos]
**Próximos passos sugeridos:** [o que fazer com a identidade agora]

Entregue:

**1. Documento completo (ONBOARDING ou ENTREGA ou ambos)**
Formatado e pronto para enviar — em tom da marca do designer.

**2. Mensagem de WhatsApp de envio**
Para acompanhar o documento quando for enviar ao cliente.

**3. FAQ pós-entrega**
As 5 perguntas mais comuns depois de receber o projeto — com respostas prontas.
```

## Exemplo de uso

### Input
- O que preciso: Entrega final
- Cliente: Nutrir Clínica
- Entregando: Logo (primária + variações horizontais e verticais), paleta oficial, tipografia, cartão de visita, templates de stories (5 layouts), mini-manual de marca
- Onde: Google Drive — link compartilhado
- Arquivos: Organizados em pastas: Logo / Aplicações / Manual / Fontes
- Formatos: AI (edição), SVG (web e impressão profissional), PNG transparente (uso geral), PDF (manual)
- Pode/não pode: Pode alterar tamanho / Não pode alterar cores / Não pode distorcer proporções
- Próximos passos: Usar templates de stories desde hoje / Levar o cartão para gráfica / Compartilhar o manual com quem criar conteúdo

### Output
"Nutrir Clínica 🌿 Chegou o dia!

Sua identidade visual está pronta e organizada no link abaixo. Preparei um documento de entrega completo que explica cada arquivo, como usar cada formato e o que fazer agora com a marca nova.

📁 Link: [link]
📄 Manual de entrega: [em anexo]

Sugiro começar pelos templates de stories hoje mesmo — já estão prontos para usar.

Foi um prazer trabalhar nesse projeto. Qualquer dúvida sobre os arquivos, me chama aqui.

Um abraço,
Thiago"
