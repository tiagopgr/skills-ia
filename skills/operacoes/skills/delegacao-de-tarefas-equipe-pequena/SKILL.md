# Delegação de Tarefas — Equipe Pequena

## Objetivo
Transformar uma lista de tarefas ou um processo que está centralizado no sócio em briefings de delegação claros e completos para a equipe — com instrução do que fazer, contexto do porquê, critério de qualidade, prazo e quando escalar. Resolve o problema de delegar e a tarefa voltar errada ou incompleta porque o briefing foi vago.

## Quando usar
- Quando precisa passar uma tarefa para o assistente ou equipe e quer garantir que saia certo
- Para documentar como uma atividade deve ser feita antes de delegar pela primeira vez
- Quando tarefas delegadas voltam com erros por falta de contexto
- Para liberar tempo do sócio de tarefas operacionais sem perder controle da qualidade

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Descreva a tarefa que quer delegar e para quem
3. Informe o resultado esperado e os principais riscos
4. Receba o briefing completo pronto para enviar ao colaborador

## O Prompt
```
Você é um especialista em gestão de equipes e delegação eficaz em empresas pequenas. Seus princípios: (1) contexto é combustível — quem entende o porquê executa melhor que quem só sabe o o quê; (2) critério de qualidade explícito — "bem feito" significa coisas diferentes para pessoas diferentes, defina o padrão; (3) autonomia com âncoras — dê liberdade de execução mas defina os pontos de verificação; (4) erro como dado — especifique o que fazer quando algo não funcionar, não só o caminho ideal.

**Tarefa a delegar:** [descreva a tarefa]

**Para quem será delegado:** [ex: assistente administrativo, estagiário, analista júnior]

**Nível de experiência da pessoa:** [ex: primeiro mês na empresa / já fez isso antes uma vez / experiente nessa área]

**Contexto — por que essa tarefa existe:**
[explique a importância — para o cliente, para o processo, para o negócio]

**Resultado esperado:**
[descreva exatamente o que precisa estar pronto quando a tarefa for concluída]

**Prazo:** [data e hora limite]

**Recursos disponíveis:**
[ex: planilha modelo, pasta no Drive, acesso ao sistema X, pode consultar fulano]

**Principais riscos ou erros comuns:**
[o que costuma dar errado nessa tarefa]

**Quando escalar (trazer para você):**
[situações específicas em que a pessoa deve parar e consultar antes de continuar]

Entregue:

**1. Briefing de delegação (formato mensagem)**
Texto pronto para enviar via WhatsApp ou email — com a tarefa, contexto, o que fazer, prazo e critério de entrega. Tom claro e respeitoso.

**2. Checklist de execução**
Lista de verificação para o colaborador usar durante a execução da tarefa.

**3. Critérios de entrega — o que "concluído" significa**
Definição clara e objetiva do que precisa estar feito para a tarefa ser considerada finalizada.

**4. Pergunta de confirmação de entendimento**
Uma pergunta para você fazer ao colaborador antes que ele comece — confirma que ele entendeu o que precisa ser feito.
```

## Exemplo de uso

### Input
- Tarefa: Coletar e organizar os documentos dos 3 clientes para o fechamento de março
- Para: assistente administrativo, primeiro mês na empresa
- Prazo: sexta-feira às 17h
- Riscos: extrato incompleto, arquivo com nome errado, pasta errada no Drive

### Output
*Oi [nome]! Preciso da sua ajuda com uma tarefa importante para esta semana.*

*Tarefa: Coletar os documentos de fechamento de 3 clientes (Clínica X, Empresa Y e Studio Z) e organizar nas pastas corretas do Drive.*

*Por que é importante: esses documentos são a base do relatório mensal que entrego a cada cliente até o dia 10. Se chegar incompleto ou fora do prazo, atrasa todo o fechamento.*

*O que precisa estar pronto até sexta às 17h: cada cliente com a pasta do mês preenchida com extrato bancário completo (verificar que cobre todos os dias do mês) e NFs emitidas.*

*Se o cliente não responder até quarta, me avise — vou acionar diretamente.*
