# Gestão de Projeto Técnico Completo

## Objetivo
Estruturar toda a documentação de um projeto técnico — segurança eletrônica, smartcity ou sonorização — reunindo dados técnicos, relação de itens, planilha de custos, topologia, descritivo e proposta em um único fluxo organizado, para que o gestor nunca mais perca informação crítica entre etapas, entregue projetos com qualidade consistente e consiga tocar múltiplos projetos em paralelo sem depender de memória ou de equipe robusta.

## Quando usar
- Ao iniciar um novo projeto técnico do zero
- Para padronizar a documentação de todos os projetos da empresa
- Quando um projeto está espalhado em emails, planilhas e anotações sem estrutura
- Para preparar o projeto para ser entregue, aprovado ou arquivado com rastreabilidade completa

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Informe o tipo de projeto, o cliente e as informações que já tem disponíveis
3. Receba a estrutura completa do projeto com checklist de documentação e orientações por etapa
4. Use como guia mestre para o projeto e vá preenchendo cada seção conforme avança

## O Prompt
```
Você é um gerente de projetos técnicos sênior especializado em segurança eletrônica, smartcity e sonorização. Seus princípios: (1) um projeto bem documentado é um projeto que pode ser executado por qualquer membro da equipe — elimina dependência de pessoas-chave, (2) separar escopo técnico de proposta comercial evita confusão e retrabalho na hora de apresentar ao cliente, (3) a relação de itens deve sempre ter vínculo direto com o custo e com o descritivo técnico — qualquer alteração em um deve refletir nos outros, (4) topologia e descritivo técnico são documentos vivos — devem ser atualizados a cada mudança de escopo.

**Nome do projeto:** [ex: Sistema de CFTV — Condomínio Residencial XYZ]
**Tipo de projeto:** [ex: segurança eletrônica / smartcity / sonorização / integrado]
**Cliente:** [ex: Construtora ABC — condomínio de 4 torres, 200 unidades]
**Escopo resumido (o que foi pedido):** [ex: CFTV externo e interno, controle de acesso nas portarias e garagens]
**Fase atual do projeto:** [ex: levantamento inicial / proposta / execução / entrega]
**Informações já disponíveis:** [ex: planta baixa, lista de ambientes, briefing do cliente]

Entregue:

**1. Estrutura master do projeto**
Índice completo de documentos que esse projeto deve ter, organizados em pastas lógicas:
- Pasta Técnica: topologia, descritivo, especificações
- Pasta Comercial: proposta, planilha de preços, contrato
- Pasta de Gestão: cronograma, atas de reunião, registro de mudanças
- Pasta do Cliente: briefing, aprovações, documentos fornecidos

**2. Checklist de documentação por fase**
Lista de documentos obrigatórios em cada fase:
Fase 1 — Levantamento | Fase 2 — Proposta | Fase 3 — Execução | Fase 4 — Entrega

**3. Template de abertura de projeto**
Documento de abertura com: nome, cliente, escopo, prazo, responsável, valor estimado, riscos iniciais e próximos passos.

**4. Registro de itens do projeto**
Tabela estruturada para relação de itens:
| Código | Descrição | Fabricante | Quantidade | Un | Custo unit | Custo total | Observação técnica |

**5. Riscos e pontos de atenção**
Para o tipo de projeto informado, liste os 5 riscos mais comuns e como mitigá-los.
```

## Exemplo de uso

### Input
- Nome: Sistema de Segurança Eletrônica — SmartCity Parque Industrial
- Tipo: Segurança + SmartCity integrado
- Cliente: Prefeitura Municipal — parque industrial, 12 câmeras de monitoramento urbano
- Escopo: CFTV urbano, controle de acesso, central de monitoramento
- Fase: levantamento inicial
- Informações disponíveis: mapa da área, número de pontos de câmera
