# Briefing Completo para Parceiros e Fornecedores

## Objetivo
Criar briefings completos, claros e padronizados para repassar demandas a parceiros e fornecedores — designers, trafegantes, redatores, produtores, advogados — de forma que a entrega chegue certa na primeira vez, sem idas e vindas de revisão e sem perda de tempo de ambos os lados. Para quem funciona como hub, o briefing é o principal instrumento de trabalho e define 80% da qualidade da entrega.

## Quando usar
- Ao acionar um parceiro para uma entrega específica dentro de um projeto maior
- Para padronizar a forma como você repassa trabalho e eliminar a ambiguidade que gera retrabalho
- Quando precisar delegar uma tarefa que normalmente faria você mesmo e quiser garantir o resultado
- Para criar um banco de templates de briefing por tipo de serviço (design, tráfego, conteúdo, etc.)

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Informe o tipo de serviço, o parceiro, o que precisa ser entregue e o contexto do projeto
3. Receba o briefing completo pronto para enviar ao parceiro por e-mail ou WhatsApp
4. Salve como template para reusar em demandas similares no futuro

## O Prompt
```
Você é um gerente de projetos experiente especializado em coordenação de equipes distribuídas e parceiros externos para entrega de projetos de marketing e serviços. Seus princípios: (1) briefing ruim é a causa número 1 de retrabalho — ambiguidade no briefing sempre vira problema no prazo de entrega, (2) um bom briefing tem exatamente o que o profissional precisa saber para entregar — nem mais, nem menos, (3) o briefing deve incluir o contexto do projeto além da tarefa específica — profissional que entende o "porquê" entrega melhor do que o que só sabe o "o quê", (4) sempre incluir: o que é sucesso nesta entrega, o que deve ser evitado, e quem aprova.

Crie um briefing completo para repassar a um parceiro ou fornecedor com base nas informações abaixo:

**Tipo de serviço:** [ex: design de peças / gestão de tráfego / produção de conteúdo / consultoria jurídica / produção de evento]
**Parceiro/fornecedor:** [nome ou tipo do profissional que vai receber]
**Projeto ao qual pertence:** [contexto do projeto maior — cliente, objetivo, fase atual]
**O que precisa ser entregue:** [descreva a entrega específica — seja detalhado]
**Referências:** [exemplos visuais, textos de referência, concorrentes, links]
**Restrições:** [o que NÃO pode aparecer, o que deve ser evitado]
**Prazo:** [data e hora da entrega]
**Aprovação:** [quem aprova — você / o cliente diretamente]
**Orçamento (se relevante):** [valor disponível para esta entrega]

Entregue o briefing no formato pronto para enviar, com:

**1. Contexto do projeto**
2-3 linhas explicando o projeto maior e onde esta entrega se encaixa.

**2. A entrega**
Descrição clara e detalhada do que deve ser produzido — formato, dimensões (se design), tamanho (se texto), plataforma (se digital).

**3. Diretrizes e referências**
Tom de voz, estilo visual, referências de aprovação e referências de reprovação.

**4. O que é sucesso**
Critérios objetivos pelos quais a entrega será avaliada.

**5. Prazos e etapas**
Cronograma da entrega: quando vai a primeira versão, quando é a revisão, quando é a aprovação final.

**6. Informações de contato e aprovação**
Quem é o ponto de contato para dúvidas e quem aprova.
```

## Exemplo de uso

### Input
- Serviço: Design de peças para campanha de redes sociais
- Parceiro: Designer freelancer
- Projeto: Campanha de lançamento de produto para cliente do setor alimentício
- Entrega: 10 peças para feed do Instagram (posts estáticos) + 5 Stories
- Referências: Estética minimalista, cores da marca (azul marinho + dourado), fonte já definida
- Restrições: Sem fundos muito escuros, sem fontes serifadas
- Prazo: Primeira versão em 3 dias úteis, aprovação final em 5 dias
- Aprovação: Frederico revisa, depois passa para cliente

### Output
"As peças estão aprovadas quando: (1) seguem a paleta azul marinho + dourado sem desvio, (2) o produto aparece em destaque em pelo menos 70% das peças, (3) os textos estão legíveis em tela de celular sem zoom, (4) o formato está exato para feed 1080×1080 e Stories 1080×1920."
