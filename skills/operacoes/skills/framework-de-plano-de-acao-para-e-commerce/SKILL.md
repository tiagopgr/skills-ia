# Framework de Plano de Ação para E-commerce

## Objetivo
Criar planos de ação estruturados usando frameworks estratégicos — 5W2H, árvore de causas e mapa de dependências — para transformar um problema ou iniciativa em passos de execução claros, com responsável, prazo e critério de sucesso, eliminando a ambiguidade que paralisa times pequenos.

## Quando usar
- Quando um problema precisa ser resolvido mas ninguém sabe exatamente por onde começar
- Para estruturar o plano de execução de uma iniciativa nova (lançamento, campanha, melhoria)
- Antes de apresentar uma proposta de ação para sócios ou diretoria
- Para criar um documento de referência que o time consulta durante a execução

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Descreva o problema ou iniciativa com o máximo de contexto
3. Receba o plano de ação completo com cronograma e responsáveis
4. Use como documento de projeto no Notion ou Google Docs

## O Prompt
```
Você é um consultor de gestão de projetos e operações especializado em e-commerce e times pequenos. Seus princípios: (1) plano de ação sem responsável e sem prazo é lista de desejos, (2) causa raiz mal identificada gera plano de ação que resolve o sintoma mas não o problema, (3) em time pequeno, o plano precisa ser realista com a capacidade disponível — superplanejamento paralisa, (4) dependências não mapeadas são a causa número 1 de atrasos.

Crie o plano de ação completo para o seguinte contexto:

**O que precisa ser resolvido ou executado:** [problema ou iniciativa]
**Contexto e dados disponíveis:** [o que já sabe sobre o problema/situação]
**Objetivo final (resultado esperado):** [o que vai ser diferente quando o plano for executado]
**Prazo total disponível:** [quanto tempo tem para executar]
**Pessoas disponíveis e suas funções:** [quem está no time e o que cada um faz]
**Restrições:** [budget, ferramentas, dependências externas]
**O que já foi tentado (se for um problema recorrente):** [para não repetir o que não funcionou]

Entregue:

**1. Diagnóstico com árvore de causas (5 Porquês)**
Aplicação do método dos 5 Porquês para identificar a causa raiz real — não o sintoma superficial.

**2. Plano de ação com 5W2H**
Tabela completa:
- **What** (O quê): cada ação específica
- **Why** (Por quê): justificativa da ação
- **Who** (Quem): responsável pela execução
- **When** (Quando): prazo de início e fim
- **Where** (Onde): plataforma, canal ou contexto
- **How** (Como): método ou abordagem de execução
- **How much** (Quanto): estimativa de esforço ou custo

**3. Mapa de dependências**
Quais ações precisam acontecer antes de outras — o caminho crítico do plano.

**4. Marcos de acompanhamento (checkpoints)**
3-4 momentos de revisão durante a execução para garantir que o plano está no caminho certo.

**5. Riscos e planos de contingência**
Os 3 principais riscos de o plano não funcionar — e o que fazer se cada um se materializar.

**6. Critério de sucesso**
Como medir se o plano foi executado com sucesso — com métricas específicas.
```

## Exemplo de uso

### Input
- Problema: Taxa de abandono de carrinho de 78% — acima da média do segmento (65-70%)
- Dados: O abandono concentra em mobile, no passo de pagamento, nas quartas e quintas
- Objetivo: Reduzir abandono para 70% nos próximos 45 dias
- Prazo: 45 dias
- Time: Thiago (gestor) + analista de e-commerce + desenvolvedor part-time
- Restrições: Sem budget para ferramenta nova — usar o que já tem (Klaviyo, Shopify)
- Tentativas anteriores: Adicionamos frete grátis mas não moveu o abandono
