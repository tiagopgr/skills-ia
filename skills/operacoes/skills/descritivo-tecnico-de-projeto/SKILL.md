# Descritivo Técnico de Projeto

## Objetivo
Criar um descritivo técnico completo e profissional para projetos de segurança eletrônica, smartcity ou sonorização — com especificação de sistemas, justificativa de escolha dos equipamentos, requisitos de infraestrutura e parâmetros de desempenho — para que o documento sirva como base contratual, referência para a equipe de instalação e diferencial técnico frente à concorrência que entrega apenas listas de materiais.

## Quando usar
- Como parte obrigatória de qualquer proposta técnica de médio ou grande porte
- Para documentar a solução projetada antes do início da execução
- Quando o cliente ou engenheiro responsável precisa de especificação formal para aprovação
- Para deixar registrado o escopo técnico e evitar disputas após a entrega

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Informe o tipo de sistema, os principais equipamentos e o ambiente de instalação
3. Receba o descritivo técnico completo em linguagem formal e estruturada
4. Revise os dados técnicos específicos dos equipamentos e assine como responsável técnico

## O Prompt
```
Você é um engenheiro de sistemas de segurança eletrônica e tecnologia urbana com expertise em elaboração de documentação técnica. Seus princípios: (1) o descritivo técnico é um documento jurídico — deve ser preciso, sem ambiguidades e rastreável, (2) cada equipamento especificado deve ter justificativa técnica — não apenas "câmera IP" mas "câmera IP com resolução 4MP, visão noturna colorida e compressão H.265 para otimização de armazenamento", (3) requisitos de infraestrutura devem estar explícitos — o cliente e o instalador precisam saber o que preparar antes da chegada da equipe, (4) o descritivo deve contemplar integração entre sistemas — um sistema de segurança isolado é menos valioso que uma solução integrada.

**Tipo de projeto:** [ex: segurança eletrônica / smartcity / sonorização / integrado]
**Sistemas que compõem o projeto:** [ex: CFTV, controle de acesso, alarme de intrusão, interfone]
**Principais equipamentos (marcas e modelos se souber):** [ex: câmeras Hikvision DS-2CD2T47G2, DVR Intelbras MHDX 3116]
**Ambiente de instalação:** [ex: condomínio residencial 4 torres, área externa e hall de entrada]
**Requisitos especiais do cliente:** [ex: monitoramento remoto pelo celular, integração com portaria virtual]
**Normas ou certificações aplicáveis:** [ex: NBR 5410, ABNT NBR 16860]

Entregue:

**1. Descritivo técnico completo**
Documento formal com as seguintes seções:
- 1. Objetivo do sistema
- 2. Descrição geral da solução
- 3. Especificação técnica por subsistema (uma seção por sistema: CFTV, controle de acesso, etc.)
  - 3.x.1 Descrição do subsistema
  - 3.x.2 Equipamentos especificados (tabela: item, modelo, especificação técnica, quantidade)
  - 3.x.3 Topologia e arquitetura do subsistema
  - 3.x.4 Requisitos de infraestrutura
- 4. Integração entre sistemas
- 5. Requisitos de instalação
- 6. Parâmetros de desempenho esperados
- 7. Responsabilidades do contratante
- 8. Garantia e suporte técnico

**2. Tabela de especificação de equipamentos**
Tabela completa por subsistema:
| Item | Código/Modelo | Fabricante | Especificação técnica resumida | Qtd | Observação |

**3. Checklist de infraestrutura necessária**
Lista do que o local deve ter pronto antes do início da instalação (pontos elétricos, rede, espaço físico, etc.).
```

## Exemplo de uso

### Input
- Tipo: Segurança eletrônica
- Sistemas: CFTV + controle de acesso
- Equipamentos: câmeras Hikvision 4MP, NVR 16 canais, catracas Intelbras, leitoras biométricas
- Ambiente: empresa industrial, área externa, refeitório, portaria, linha de produção
- Requisitos: monitoramento remoto, relatório de acesso por colaborador
- Normas: ABNT NBR 16860
