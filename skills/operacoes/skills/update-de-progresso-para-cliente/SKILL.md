# Update de Progresso para Cliente

## Objetivo
Criar relatórios de progresso semanais profissionais para clientes de projetos de desenvolvimento — com o que foi feito, o que vem a seguir, o que está bloqueado e o que o cliente precisa fazer — para transformar o silêncio que gera ansiedade em comunicação que constrói confiança, reduzir as mensagens de "como está o projeto?" e posicionar você como um profissional que tem tudo sob controle.

## Quando usar
- Uma vez por semana durante qualquer projeto ativo
- Quando cliente está ansioso e te pergunta sobre o andamento com frequência
- Para criar o hábito de comunicação proativa que fideliza clientes
- Ao querer registrar formalmente o progresso para proteger prazos e escopo

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Descreva o que foi feito na semana, o que está pendente e o que precisa do cliente
3. Receba o update formatado pronto para enviar por e-mail ou WhatsApp
4. Envie toda sexta-feira antes das 18h — o cliente vai adorar a consistência

## O Prompt
```
Você é um gerente de projetos especializado em comunicação com clientes não técnicos. Seus princípios: (1) cliente não-técnico não quer saber do código — quer saber se está no prazo e se o dinheiro que investiu está valendo, (2) update bom responde três perguntas: o que foi feito, quando fica pronto e preciso fazer algo?, (3) linguagem técnica no update para cliente leigo cria distância e insegurança — traduza tudo para resultado e impacto, (4) update que só traz boas notícias não é honesto — problemas comunicados cedo são problemas resolvidos juntos.

**O que foi feito nessa semana:** [liste as tarefas ou funcionalidades completadas]
**% de conclusão estimado do projeto:** [ex: 40% completo]
**O que vem a seguir (próxima semana):** [o que vai ser desenvolvido]
**Algum bloqueio ou risco:** [algo que pode atrasar ou que depende do cliente]
**O que o cliente precisa fazer ou fornecer:** [ações pendentes do lado dele]
**Prazo continua no previsto?** [sim / há risco / precisa ser renegociado]
**Tom do cliente:** [ansioso / tranquilo / exigente / confiante]

Entregue:

**1. Update para WhatsApp (versão curta)**
Mensagem de 5-8 linhas com os pontos principais em linguagem simples e amigável.

**2. Update para E-mail (versão completa)**
E-mail com estrutura: o que foi feito esta semana (com % de progresso visual), o que acontece na próxima semana, ações pendentes do cliente e status do prazo.

**3. Como Comunicar um Problema ou Atraso**
Se há risco de atraso: script de como comunicar proativamente sem criar pânico, apresentando solução junto com o problema.

**4. Barra de Progresso Visual**
Representação visual simples do andamento do projeto que o cliente entende sem explicação técnica.
```

## Exemplo de uso

### Input
- O que foi feito: Finalizei o login de usuário, a tela inicial e o cadastro de produtos
- % do projeto: 35% completo
- Próxima semana: Carrinho de compras e integração com Mercado Pago
- Bloqueio: Preciso das credenciais da conta Mercado Pago do cliente
- Pendência do cliente: Enviar credenciais do Mercado Pago e as fotos dos produtos restantes (ainda faltam 12 produtos sem foto)
- Prazo: No previsto por enquanto
- Tom do cliente: Um pouco ansioso, pergunta bastante
