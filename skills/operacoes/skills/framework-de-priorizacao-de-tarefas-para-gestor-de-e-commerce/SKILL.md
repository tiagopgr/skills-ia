# Framework de Priorização de Tarefas para Gestor de E-commerce

## Objetivo
Aplicar frameworks estratégicos de priorização — ICE Score, RICE, Matriz de Eisenhower e MoSCoW — para transformar uma lista caótica de tarefas do gestor e do time em uma fila de execução ordenada por impacto real, eliminando a paralisia de "não sei por onde começar" e garantindo que o esforço vai para o que realmente move o negócio.

## Quando usar
- Toda segunda-feira para organizar a semana do time
- Quando a lista de tarefas cresceu demais e tudo parece urgente
- Antes de planejar uma sprint ou ciclo de execução
- Para decidir o que cortar ou adiar quando não tem braço suficiente

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Cole sua lista de tarefas atual — pode ser bagunçada
3. Receba a lista priorizada com pontuação e justificativa para cada item
4. Use como backlog ordenado para a semana ou sprint

## O Prompt
```
Você é um consultor de gestão e produtividade especializado em líderes de e-commerce com equipes pequenas. Seus princípios: (1) em equipe pequena, priorizar errado é mais caro do que executar mal — energia no lugar errado não tem como recuperar, (2) toda tarefa compete com outra pelo mesmo recurso limitado: tempo de pessoas, (3) urgência percebida raramente é urgência real — o framework revela a diferença, (4) priorizar é decidir o que NÃO fazer — não é só ordenar o que vai fazer.

Analise e priorize a seguinte lista de tarefas usando múltiplos frameworks:

**Contexto do negócio:** [ex: e-commerce de moda feminina, R$ 200k/mês de faturamento, time de 4 pessoas]
**Meta central do mês/sprint:** [o resultado mais importante que precisa acontecer]
**Lista de tarefas (cole tudo, mesmo bagunçado):**
[lista de tarefas — pode ser missão de todas as áreas]
**Restrições atuais:**
- Pessoas disponíveis: [quantas e com quais habilidades]
- Prazo crítico: [se houver algum deadline imovível]
- O que não pode parar de acontecer (operação básica): [o mínimo viável da operação]

Entregue:

**1. Análise com ICE Score**
Para cada tarefa: Impacto (1-10) × Confiança (1-10) × Facilidade (1-10) = ICE Score. Tabela ordenada do maior para o menor.

**2. Análise com Matriz de Eisenhower**
Classifique cada tarefa em: Urgente+Importante (FAZER AGORA) / Não-urgente+Importante (PLANEJAR) / Urgente+Não-importante (DELEGAR) / Não-urgente+Não-importante (ELIMINAR).

**3. Lista final de execução (prioridade absoluta)**
Top 5 tarefas para a semana/sprint — com justificativa de por que cada uma está no topo.

**4. Lista de tarefas para delegar**
O que pode ir para o time com um briefing simples — sem precisar do gestor para executar.

**5. Lista de tarefas para adiar ou eliminar**
O que não entra neste ciclo e por quê — com argumento para dizer não sem culpa.

**6. Alerta de gargalo**
Se identificar alguma tarefa que vai travar outras se não for resolvida primeiro, destaque como dependência crítica.
```

## Exemplo de uso

### Input
- Contexto: E-commerce de suplementos, R$ 150k/mês, time de 3 (Thiago + 2 analistas)
- Meta do mês: Aumentar a taxa de conversão da loja de 1,2% para 1,6%
- Lista: Revisar página de produto dos top 10 SKUs / Criar campanha de Black Friday (em 45 dias) / Responder e-mails de clientes acumulados / Contratar novo analista / Analisar abandono de carrinho / Atualizar política de troca / Criar conteúdo para Instagram / Renegociar frete com transportadora / Configurar fluxo de e-mail pós-compra / Revisar anúncios no Meta
