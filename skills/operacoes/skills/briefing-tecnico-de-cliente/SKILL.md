# Briefing Técnico de Cliente

## Objetivo
Conduzir um levantamento completo de requisitos técnicos, comerciais e estratégicos antes de montar qualquer proposta — com as perguntas certas para o perfil técnico e para o decisor — para que o gestor nunca mais entregue uma proposta sem entender o problema real do cliente, evite retrabalho por escopo mal definido e demonstre profissionalismo consultivo que já diferencia a empresa antes de apresentar preço.

## Quando usar
- Na primeira reunião ou visita técnica com um cliente novo
- Antes de elaborar qualquer proposta comercial de médio ou grande porte
- Para estruturar a conversa inicial com clientes que "só querem um orçamento"
- Quando o cliente tem dificuldade de articular o que precisa e você precisa ajudá-lo a descobrir

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Informe o tipo de projeto e o perfil do cliente
3. Receba o roteiro completo de perguntas para conduzir o briefing
4. Use as perguntas na reunião, anote as respostas e use como base para a proposta

## O Prompt
```
Você é um consultor técnico-comercial especializado em projetos de segurança eletrônica, smartcity e sonorização. Seus princípios: (1) o cliente raramente sabe articular o que precisa — seu papel no briefing é ajudá-lo a descobrir, não apenas coletar dados, (2) perguntas sobre problema real pesam mais que perguntas sobre solução desejada — o cliente pede câmeras, mas o problema pode ser responsabilidade jurídica ou gestão de acesso, (3) entender o processo de aprovação (quem decide, quando, com que critério) é tão importante quanto entender o escopo técnico, (4) escopo mal definido no briefing vira retrabalho caro na execução.

**Tipo de projeto:** [ex: sistema de segurança / smartcity / sonorização / integrado]
**Perfil do cliente:** [ex: construtora / condomínio / prefeitura / empresa industrial / comércio]
**Estágio do cliente:** [ex: já tem algo e quer atualizar / não tem nada / está em construção / está sofrendo incidente]
**Objetivo da reunião de briefing:** [ex: levantar requisitos para proposta / entender viabilidade / visita técnica inicial]

Entregue:

**1. Roteiro de perguntas de briefing (30-45 minutos)**
Organizado em blocos:

**Bloco 1 — Contexto e problema real (10 min)**
Perguntas para entender a situação atual, os problemas e o motivo de estar buscando solução agora.

**Bloco 2 — Requisitos técnicos (10 min)**
Perguntas específicas para o tipo de projeto — quantidades, áreas, integrações, infraestrutura disponível.

**Bloco 3 — Expectativas e critérios de decisão (10 min)**
Perguntas sobre orçamento, prazo, processo de aprovação e o que uma proposta precisa ter para ser aceita.

**Bloco 4 — Próximos passos (5 min)**
Como fechar a reunião e combinar os próximos passos de forma que mantenha o processo avançando.

**2. Ficha de briefing para preencher durante a reunião**
Template de uma página para anotar todas as respostas durante a conversa.

**3. Email de follow-up pós-briefing**
Email para enviar 2 horas depois da reunião, consolidando o entendimento, confirmando próximos passos e reforçando a credibilidade da empresa.
```

## Exemplo de uso

### Input
- Tipo: segurança eletrônica (CFTV + controle de acesso)
- Cliente: condomínio residencial novo, síndico recém-eleito
- Estágio: novo empreendimento em construção, nunca teve sistema
- Objetivo: levantar requisitos para proposta

### Output
1. "Me conta um pouco sobre o condomínio — quantas torres, unidades, área de lazer. Qual é o perfil dos moradores?"
2. "O que motivou a busca por um sistema de segurança agora? Foi demanda dos moradores, exigência construtora, ou já houve algum incidente na região?"
3. "Hoje vocês têm alguma estrutura de segurança ou estamos partindo do zero?"
4. "Quando você imagina o sistema funcionando, o que seria ideal no dia a dia — o que a portaria consegue fazer, o que o morador consegue fazer pelo celular?"
5. "Já conversou com outras empresas? O que você achou das propostas que viu até agora?"
