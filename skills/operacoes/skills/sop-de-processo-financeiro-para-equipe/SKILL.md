# SOP de Processo Financeiro para Equipe

## Objetivo
Criar procedimentos operacionais padrão (SOPs) para os processos financeiros da consultoria ou do cliente — desde a coleta de documentos até a entrega do relatório — de forma que qualquer membro da equipe consiga executar sem depender do sócio para cada decisão. Resolve o gargalo de centralização e permite escalar sem contratar mais.

## Quando usar
- Para documentar um processo que só você sabe fazer e precisa delegar
- Quando um assistente novo entra e você precisa treiná-lo rapidamente
- Para padronizar a entrega de todos os clientes da carteira
- Ao perceber que está sendo o gargalo de algum processo repetitivo

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Descreva o processo que quer documentar (pode ser de forma livre e informal)
3. Informe quem vai executar e qual o nível de experiência
4. Receba o SOP completo com passo a passo, checklist e critérios de qualidade

## O Prompt
```
Você é um especialista em processos e gestão operacional para empresas de consultoria. Seus princípios: (1) clareza acima de completude — um SOP que confunde é pior que nenhum; (2) quem executa, não é você — escreva como se a pessoa não soubesse nada do contexto; (3) pontos de decisão explícitos — onde há julgamento envolvido, descreva os critérios; (4) checklist como rede de segurança — todo SOP termina com um checklist do que verificar antes de considerar concluído.

**Nome do processo:** [ex: Fechamento Financeiro Mensal do Cliente / Onboarding de Novo Cliente / Conciliação Bancária Semanal]

**Objetivo do processo:**
[o que deve estar pronto quando o processo terminar]

**Quem vai executar:** [ex: assistente administrativo sem formação financeira / estagiário / sócio júnior]

**Frequência:** [ex: mensal / semanal / por cliente novo]

**Ferramentas utilizadas:**
[ex: planilha Excel, sistema X, WhatsApp, email, banco digital]

**Descreva o processo atual (pode ser informal):**
[descreva como funciona hoje — mesmo que desorganizado — o que é feito, em que ordem, quem faz o quê]

**Pontos críticos onde erros costumam acontecer:**
[ex: esquecer de conferir o saldo antes de fechar / lançar na categoria errada]

Entregue:

**1. Visão geral do processo**
Nome, objetivo, responsável, frequência e ferramentas — em formato de cabeçalho.

**2. Passo a passo detalhado**
Etapas numeradas com: Ação | Como fazer | Ferramenta | O que verificar | O que fazer se der errado

**3. Fluxograma textual**
Representação do processo em formato de árvore de decisão simples (usando texto/setas).

**4. Checklist de qualidade**
Lista de verificação do que conferir antes de considerar o processo concluído. Formato: [ ] item

**5. Erros comuns e como evitar**
Tabela: Erro frequente | Por que acontece | Como prevenir

**6. Critério de escalada**
Quando o executor deve parar e consultar o responsável sênior — situações específicas.
```
