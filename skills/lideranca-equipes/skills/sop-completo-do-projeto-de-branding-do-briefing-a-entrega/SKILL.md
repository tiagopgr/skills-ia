# SOP Completo do Projeto de Branding — Do Briefing à Entrega

## Objetivo
Documentar o processo completo de um projeto de branding — do primeiro contato ao arquivo final entregue — em um SOP estruturado com etapas, responsabilidades, pontos de aprovação e comunicação padrão com o cliente — para que o designer execute todo projeto com consistência, sem esquecer etapas, sem surpresa de escopo e sem retrabalho por falta de processo.

## Quando usar
- Quando quiser padronizar como executa todos os projetos de branding
- Quando estiver começando a delegar partes do processo para alguém
- Quando um projeto anterior teve problema por falta de processo claro
- Quando quiser criar um material de referência para novos colaboradores
- Quando quiser profissionalizar a operação do negócio criativo

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Preencha com como você trabalha hoje e o que quer padronizar
3. Receba o SOP completo do projeto com etapas e comunicações padrão
4. Implemente nos próximos projetos e ajuste conforme necessário

## O Prompt
```
Você é um consultor de operações especializado em negócios criativos de design e branding. Seus princípios: (1) projeto criativo sem processo é roleta — o resultado depende de como o dia está, não de como o processo funciona, (2) o que o cliente não sabe que vai acontecer vai virar expectativa frustrada — comunicação proativa é parte do processo, não extra, (3) ponto de aprovação claro protege o designer de "mas eu não aprove isso" — cada etapa precisa de sign-off formal, (4) checklist antes de entregar é respeito pelo próprio trabalho — nunca entregue sem revisar.

Crie o SOP completo do projeto de branding com as seguintes informações:

**Tipo de projeto que vai documentar:** [ex: identidade visual completa / logo + manual / rebrand]
**Como funciona hoje (descreva livremente):** [o que você já faz — mesmo que sem processo formal]
**Ferramentas que usa:** [ex: Figma, Notion, e-mail, WhatsApp, WeTransfer, Dropbox]
**Quantas rodadas de revisão inclui:** [padrão do seu processo]
**Como comunica com o cliente hoje:** [e-mail, WhatsApp, reunião de apresentação]
**Onde costuma ter problema:** [onde o projeto trava, atrasa ou gera conflito]
**Resultado final entregue:** [quais arquivos, em quais formatos]

Entregue:

**1. SOP completo fase a fase**

FASE 1 — Comercial e Onboarding
FASE 2 — Briefing e Pesquisa
FASE 3 — Conceito e Criação
FASE 4 — Apresentação e Aprovação
FASE 5 — Refinamento
FASE 6 — Entrega e Fechamento

Para cada fase: passo a passo numerado, ferramenta usada, responsável, tempo estimado, resultado esperado ao final.

**2. Checklist de entrega final**
O que verificar antes de enviar qualquer arquivo para o cliente — qualidade, formato, organização.

**3. Templates de comunicação por fase**
Mensagem padrão para: início do projeto / envio de apresentação / solicitação de aprovação / aviso de entrega / encerramento.

**4. Protocolo de revisão**
Como conduzir rodadas de revisão sem perder o controle do escopo — o que é revisão, o que é novo pedido, como cobrar por extra.

**5. O que fazer quando o projeto trava**
Playbook para as 3 situações mais comuns: cliente some / cliente pede alteração fora do escopo / cliente rejeita tudo.
```

## Exemplo de uso

### Input
- Projeto: Identidade visual completa (logo + paleta + tipografia + aplicações + mini-manual)
- Hoje: Faço briefing por WhatsApp, começo criando, apresento no Figma, vou ajustando conforme o cliente pede
- Ferramentas: Figma, WhatsApp, Google Drive, e-mail
- Revisões: 2 incluídas
- Comunicação: WhatsApp principalmente
- Problemas: Cliente some durante o projeto / pede alteração infinita / acha que revisão é fazer tudo de novo
- Entrega: Figma + PDF do manual + arquivos AI/PNG/SVG

### Output
"O QUE É REVISÃO (incluído):
→ Ajuste de proporção ou espessamento do símbolo
→ Mudança de tonalidade dentro da paleta aprovada
→ Ajuste de espaçamento tipográfico
→ Refinamento de detalhes no símbolo

O QUE NÃO É REVISÃO (cobrado à parte):
→ Nova direção criativa após aprovação do conceito
→ Mudança de cor principal após aprovação de paleta
→ Adicionar elemento que não estava no briefing original
→ Refazer a logo do zero após aprovação

COMO COMUNICAR AO CLIENTE (mensagem padrão):
'[Nome], recebi seu feedback! Vou fazer os ajustes do ponto 1, 2 e 3 — que estão dentro das rodadas de refinamento que acordamos. O ponto 4 é uma direção nova que não estava no briefing original, então vou fazer um orçamento separado para esse item. Tudo bem assim?'"
