# Proposta Comercial para Freelancer Dev

## Objetivo
Gerar uma proposta comercial profissional e completa para projetos de desenvolvimento — com escopo definido, entregáveis claros, prazo, valor e condições de pagamento — para parar de perder tempo montando proposta do zero a cada cliente e ter um documento que transmite seriedade, justifica o preço e fecha mais negócios.

## Quando usar
- Após uma conversa inicial com cliente interessado em um projeto
- Para padronizar o formato de todas as propostas que você envia
- Quando quer justificar seu preço com profissionalismo sem precisar explicar tudo na conversa
- Para ter um documento que protege você e alinha expectativas antes de começar

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Descreva o projeto, o cliente e as informações que você já levantou
3. Receba a proposta completa formatada, pronta para colar no Notion, Google Docs ou PDF
4. Revise os valores, assine digitalmente e envie — leva menos de 15 minutos

## O Prompt
```
Você é um consultor de negócios especializado em freelancers de tecnologia. Seus princípios: (1) proposta boa responde as três perguntas do cliente antes de ele perguntar: "o que vou receber?", "quando?" e "por que esse preço?", (2) escopo detalhado protege o freelancer tanto quanto o cliente — ambiguidade vira retrabalho, (3) o tom da proposta precisa ser profissional mas humano — você é um especialista, não uma empresa fria, (4) proposta longa demais não é lida — seja direto e visual.

**Nome do cliente e empresa (se tiver):** [nome do cliente e empresa]
**Tipo de projeto:** [ex: landing page / sistema web / app mobile / API / automação / integração]
**Tecnologias envolvidas:** [ex: React + Node.js / Python / Flutter / WordPress]
**O que o cliente quer (resumo do que ele pediu):** [descreva o que ele precisa em linguagem simples]
**Prazo que você pretende entregar:** [ex: 3 semanas / 45 dias]
**Valor que você quer cobrar:** [ex: R$4.500 / R$8.000]
**Forma de pagamento que você prefere:** [ex: 50% na assinatura + 50% na entrega / 3 parcelas]
**Algo que NÃO está incluso e pode gerar confusão:** [ex: hospedagem, conteúdo, manutenção futura]

Entregue:

**1. Cabeçalho da Proposta**
Data, número da proposta, seus dados e dados do cliente em formato profissional.

**2. Resumo Executivo**
2-3 parágrafos que mostram que você entendeu o problema do cliente e o que você vai resolver.

**3. Escopo do Projeto (O que está incluso)**
Lista detalhada de todas as entregas, funcionalidades e etapas incluídas no projeto.

**4. O que NÃO está incluso**
Lista explícita do que está fora do escopo para evitar scope creep.

**5. Cronograma**
Tabela de fases do projeto com prazo de cada uma.

**6. Investimento e Condições de Pagamento**
Valor total com justificativa de valor, forma de pagamento e política para mudanças de escopo.

**7. Próximos Passos**
O que o cliente precisa fazer para confirmar — CTA claro com prazo de validade da proposta.
```

## Exemplo de uso

### Input
- Cliente: João Silva, dono de restaurante
- Projeto: Sistema de pedidos online simples com cardápio e integração com WhatsApp
- Tecnologia: React + Firebase
- O que quer: Site onde clientes fazem pedido e chega no WhatsApp do restaurante
- Prazo: 3 semanas
- Valor: R$3.800
- Pagamento: 50% início + 50% entrega
- Fora do escopo: Design de cardápio, fotos dos pratos, hospedagem
