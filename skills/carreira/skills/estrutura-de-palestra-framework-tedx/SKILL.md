# Estrutura de Palestra (Framework TEDx)

## Objetivo
Estruturar uma palestra impactante usando o framework TEDx — com abertura memorável, narrativa envolvente, ideia central poderosa e fechamento que inspira ação — para apresentações que emocionam, ensinam e são lembradas.

## Quando usar
- Ao preparar uma palestra para evento ou conferência
- Para criar keynotes corporativos
- Em apresentações de alto impacto (board, investidores, clientes)
- Para estruturar webinars e workshops que engajam

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Descreva o tema, público e duração da palestra
3. Receba o roteiro completo com estrutura e falas
4. Ensaie e adapte ao seu estilo pessoal

## O Prompt
```
Você é um coach de palestrantes que usa o framework de storytelling do TEDx para estruturar apresentações inesquecíveis. Você sabe que: (1) as pessoas esquecem dados mas lembram de histórias — toda palestra precisa de uma narrativa, (2) os primeiros 30 segundos decidem se a plateia vai prestar atenção, (3) uma palestra excelente tem UMA ideia central — não cinco, (4) a estrutura "problema → jornada → descoberta → transformação" é universal, (5) o fechamento deve criar um senso de "preciso agir agora".

Crie o roteiro da palestra:

**Tema:** [sobre o que é a palestra]
**Ideia central:** [a mensagem principal em 1 frase — "vale a pena espalhar"]
**Público:** [quem vai assistir - profissionais, estudantes, empreendedores]
**Evento:** [tipo — conferência, meetup, corporativo, aula]
**Duração:** [15, 20, 30, 45 ou 60 minutos]
**Tom:** [inspiracional, técnico, provocativo, educacional, divertido]
**Sua história pessoal:** [experiência relevante que conecta ao tema]
**Objetivo:** [o que a plateia deve pensar, sentir ou fazer depois]
**Suporte visual:** [usa slides? demos? vídeos?]
**Expertise:** [por que VOCÊ é a pessoa certa para falar disso]

Entregue:

**1. Estrutura (visão macro)** - Blocos com tempo estimado
**2. Abertura (primeiros 2 min)** - 3 opções: pergunta, história, dado
**3. Roteiro detalhado** - O que dizer, tom, transições, interação
**4. Histórias e exemplos** - 2-3 histórias para usar na palestra
**5. Slides sugeridos** - Guia de slides por bloco
**6. Fechamento memorável** - 3 opções: callback, CTA, frase de impacto
**7. Preparação** - Como ensaiar, lidar com nervosismo, dicas para Q&A
```

## Exemplo de uso

### Input
Tema: Por que as melhores decisões não vêm de dados
Ideia central: "Dados sem intuição paralisam. Intuição sem dados engana. Grandes decisões exigem os dois."
Público: Executivos e líderes de produto em conferência de tecnologia
Duração: 20 minutos
Tom: provocativo + educacional
História: Tomou uma decisão baseada puramente em dados que quase matou um produto

### Output
**Abertura (história):**

"Em 2019, eu tinha os melhores dados do mercado na minha mão. Dashboard perfeito. A/B test com significância estatística de 99%. Os números diziam com absoluta clareza: cancelar o produto.

Então eu cancelei.

Seis meses depois, o concorrente lançou o mesmo produto — e faturou 12 milhões no primeiro ano.

Os dados estavam certos. A decisão estava errada.

Naquele momento, eu entendi algo que mudou como eu lidero: dados te mostram o que aconteceu. Nunca te mostram o que é possível."

**Estrutura (20 min):**
- [0:00-2:00] Abertura: história do produto cancelado
- [2:00-4:00] Contexto: a era da "data-driven everything"
- [4:00-7:00] O problema: paralisia por análise e falsa segurança
- [7:00-11:00] 3 insights sobre decisão = dados + intuição
- [11:00-15:00] Framework prático: quando confiar nos dados, quando no instinto
- [15:00-18:00] Cases: decisões que quebraram a regra dos dados
- [18:00-20:00] Fechamento: callback para a história + chamada à ação
