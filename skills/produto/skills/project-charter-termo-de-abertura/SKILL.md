# Project Charter (Termo de Abertura)

## Objetivo
Criar um Project Charter completo que alinha todas as partes interessadas antes de iniciar um projeto — definindo escopo, objetivos, stakeholders, riscos e critérios de sucesso — para evitar o clássico 'não era isso que eu esperava' no meio do caminho.

## Quando usar
- Ao iniciar qualquer projeto significativo (interno ou para cliente)
- Para alinhar expectativas com stakeholders antes de começar
- Quando múltiplas áreas estão envolvidas no projeto
- Como documento de referência durante toda a execução

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Descreva o projeto e contexto
3. Receba o Project Charter completo
4. Revise com stakeholders e obtenha aprovação antes de iniciar

## O Prompt
```
Você é um gerente de projetos que sabe que projetos sem charter claro têm 2x mais chance de falhar. O charter não é burocracia — é a fundação. Ele garante que: (1) todos entendem o que o projeto vai entregar e o que NÃO vai, (2) os riscos são mapeados antes de virar problemas, (3) há um dono para cada decisão, (4) os critérios de sucesso estão definidos antes de começar.

Crie o Project Charter para:

**Nome do projeto:** [título]
**Contexto:** [por que esse projeto existe — problema ou oportunidade]
**Solicitante/Sponsor:** [quem pediu o projeto e tem poder de decisão]
**Gerente do projeto:** [quem vai liderar a execução]
**Equipe envolvida:** [áreas e pessoas]
**Prazo desejado:** [data de entrega ou duração]
**Orçamento:** [se aplicável]
**Restrições conhecidas:** [limitações de tempo, dinheiro, tecnologia, pessoas]

Estruture o charter com: Visão geral, Justificativa, Objetivos SMART, Escopo (Dentro e Fora), Entregáveis, Marcos, Stakeholders, Riscos, Premissas, Critérios de sucesso, Orçamento, Aprovação.
```

## Exemplo de uso

### Input
Projeto: Redesign do site institucional da empresa
Contexto: Site atual tem 5 anos, não é responsivo, taxa de conversão de 0.8%
Sponsor: Diretora de Marketing
Prazo: 3 meses
Equipe: Designer, dev front-end, copywriter, gestor de projeto

### Output
ESCOPO:
✅ Dentro do escopo:
- Redesign de 8 páginas (Home, Sobre, Serviços, Blog, Contato, 3 Landing Pages)
- Design responsivo (mobile + desktop)
- Nova copy para todas as páginas
- Implementação em WordPress/Webflow
- 2 rodadas de revisão por página

❌ Fora do escopo:
- Criação de conteúdo novo para blog (só migração)
- E-commerce ou funcionalidades de pagamento
- SEO técnico avançado (apenas SEO on-page básico)
- Manutenção pós-lançamento (será contratada à parte)
