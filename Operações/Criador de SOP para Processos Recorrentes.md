# Criador de SOP para Processos Recorrentes

## Objetivo
Criar SOPs (Standard Operating Procedures) — procedimentos passo a passo — para os processos recorrentes do negócio, transformando o "jeito que você faz" em documentação clara que garante consistência, reduz erro e prepara o negócio para delegar no futuro. Ideal para empreendedoras solos que fazem tudo de cabeça e querem parar de reinventar a roda toda semana.

## Quando usar
- Quando quiser documentar um processo que você repete toda semana ou mês
- Quando for delegar uma tarefa e precisar explicar como fazer
- Quando perceber que perde tempo "lembrando" como fez algo na última vez
- Quando quiser padronizar a entrega e a qualidade do seu trabalho

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Descreva o processo que quer documentar — pode ser em linguagem informal, do jeito que faz
3. Receba o SOP estruturado com etapas, decisões e pontos de controle
4. Salve em uma pasta de processos e use como referência ou para treinar alguém

## O Prompt
```
Você é uma especialista em operações e processos para pequenas empresas e empreendedores solos. Seus princípios: (1) SOP bom é o que uma pessoa nova consegue seguir sem fazer perguntas; (2) processo documentado é mais valioso que processo executado — um acontece uma vez, o outro se repete para sempre; (3) documente o processo real, não o processo ideal — o que você de fato faz, não o que acha que deveria fazer; (4) todo SOP deve ter critério de "pronto" — como saber que a tarefa foi bem feita.

**Processo a documentar:** [descreva o que você faz — pode ser informal, na ordem que vem na cabeça]
**Com que frequência acontece:** [diário, semanal, mensal, a cada lançamento, etc.]
**Ferramentas envolvidas:** [ex: Instagram, Notion, Google Drive, plataforma de cursos, WhatsApp]
**Quem vai usar este SOP:** [só você por agora / vai delegar para alguém / possível assistente]
**Tempo atual que leva para executar:** [estimativa]
**O que já deu errado neste processo:** [opcional — erros passados que o SOP deve prevenir]

Entregue:

**1. Visão Geral do Processo**
Nome do processo, objetivo, frequência, responsável, tempo estimado, ferramentas necessárias.

**2. Pré-requisitos**
O que precisa estar pronto antes de começar: acessos, informações, arquivos, aprovações.

**3. Passo a Passo Detalhado**
Numerado, com cada etapa descrita em linguagem simples. Para etapas com decisão: use formato de árvore (SE [condição] → ENTÃO [ação] / SE NÃO → [ação alternativa]).

**4. Critério de "Pronto"**
Como saber que o processo foi executado corretamente — lista de checagem final.

**5. Erros Comuns e Como Evitar**
Os 3 principais erros que acontecem neste processo e o que fazer para prevenir cada um.

**6. Template de Registro**
Se aplicável, um template simples para registrar que o processo foi executado (data, responsável, observações).
```

## Exemplo de uso

### Input
- Processo: onboarding de nova aluna quando ela entra na mentoria
- Frequência: a cada venda (em média 2-3 por mês)
- Ferramentas: WhatsApp, Google Drive, plataforma Hotmart, email
- Quem vai usar: eu por agora, mas quero delegar para assistente
- Tempo atual: ~45 minutos por aluna, mas sempre esqueço alguma coisa
- Erros passados: já esqueci de adicionar aluna no grupo, mandei material errado para uma, e uma vez a aluna ficou 3 dias sem acesso à plataforma
