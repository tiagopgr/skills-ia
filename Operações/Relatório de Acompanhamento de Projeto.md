# Relatório de Acompanhamento de Projeto

## Objetivo
Gerar relatórios de acompanhamento de projeto claros e profissionais — com resumo executivo, evolução física e financeira, próximos passos e riscos — para que o gestor mantenha o cliente informado de forma proativa, construa credibilidade durante a execução e evite surpresas e conflitos que surgem quando o cliente não sabe o que está acontecendo no projeto.

## Quando usar
- Para enviar atualizações semanais ou quinzenais ao cliente durante a execução
- Quando um marco importante foi atingido e precisa ser documentado formalmente
- Para registrar formalmente um atraso ou mudança de escopo antes que vire conflito
- Quando o cliente pede "uma atualização" e você quer entregar algo profissional ao invés de uma mensagem informal

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Informe o status atual do projeto, o que foi feito e o que está pendente
3. Receba o relatório completo formatado e pronto para enviar
4. Personalize com fotos, plantas ou dados específicos e envie ao cliente

## O Prompt
```
Você é um gerente de projetos especializado em comunicação com clientes em projetos técnicos de segurança e tecnologia. Seus princípios: (1) o relatório deve ser lido em 3 minutos — executivo no topo, detalhes abaixo, (2) nunca esconder problemas no relatório — comunicar risco com solução proposta é muito melhor do que o cliente descobrir sozinho, (3) percentual de evolução física deve ser baseado em critério objetivo, não em sensação — definir critério no início do projeto, (4) o relatório é um documento legal — em caso de disputa, ele registra o que foi entregue e quando.

**Nome do projeto:** [ex: Sistema de Segurança — Condomínio Parque Verde]
**Cliente:** [ex: Síndico João Andrade — Condomínio Parque Verde]
**Data do relatório:** [ex: 12 de maio de 2025]
**Período coberto:** [ex: semana de 05/05 a 11/05/2025]
**Fase atual:** [ex: instalação]
**Percentual de conclusão:** [ex: 65%]
**O que foi feito neste período:** [ex: instalação das câmeras externas (24 de 36), passagem de cabos torre A e B]
**O que está previsto para o próximo período:** [ex: instalação câmeras internas, configuração do NVR]
**Riscos ou impedimentos:** [ex: falta de acesso ao apartamento 304 para instalação de câmera interna]
**Marcos atingidos:** [ex: concluída passagem de cabeamento torre A]
**Marcos pendentes e prazo:** [ex: entrega final prevista para 30/05]

Entregue:

**1. Relatório completo formatado**
Documento estruturado com:
- Cabeçalho: nome do projeto, cliente, data, período, responsável
- Resumo executivo: 3-4 linhas com status geral e destaque da semana
- Evolução do projeto: barra de progresso visual + percentual
- O que foi realizado: lista detalhada com datas
- Previsto para o próximo período: lista com datas
- Marcos e cronograma: tabela com status de cada marco
- Riscos e impedimentos: identificação + impacto + ação proposta
- Solicitações ao cliente: o que é necessário da parte do cliente para avançar
- Registro fotográfico (seção para incluir fotos — espaço reservado)

**2. Email de envio do relatório**
Email curto e profissional para acompanhar o documento.

**3. Versão WhatsApp**
Resumo de 5-7 linhas para enviar via WhatsApp para clientes que preferem comunicação informal.
```

## Exemplo de uso

### Input
- Projeto: SmartCity — câmeras urbanas Parque Industrial
- Cliente: Sec. de Segurança Pública
- Período: semana de 05-11/05
- Conclusão: 40%
- Feito: instalação dos postes em 8 de 20 pontos, passagem de cabeamento óptico 60%
- Previsto: instalação dos 12 postes restantes, conexão à central
- Risco: 1 ponto de câmera bloqueado por obra de terceiros na via
