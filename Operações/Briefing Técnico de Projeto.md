# Briefing Técnico de Projeto

## Objetivo
Criar um questionário de briefing técnico completo e personalizado para levantar todos os requisitos de um projeto de desenvolvimento — funcionalidades, integrações, design, prazo, orçamento e expectativas do cliente — para entrar em qualquer projeto com clareza total, evitar surpresas no meio do caminho e ter a base para uma proposta precisa e um escopo fechado que protege o seu tempo.

## Quando usar
- Antes de enviar qualquer proposta ou estimativa de prazo
- Quando cliente chega com "quero um site / sistema / app" sem especificações
- Para padronizar como você levanta requisitos em todos os projetos
- Ao retomar um projeto antigo e precisar documentar o que foi combinado

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Descreva o tipo de projeto e o perfil do cliente
3. Receba o questionário de briefing completo pronto para enviar ao cliente
4. Envie por e-mail ou Google Forms e só parta para a proposta depois de ter as respostas

## O Prompt
```
Você é um desenvolvedor sênior com experiência em gestão de projetos. Seus princípios: (1) requisito não levantado antes vira bug depois — o briefing é o investimento mais barato do projeto, (2) cliente sem experiência técnica não sabe o que quer até ver opções — seu briefing precisa educá-lo enquanto levanta os dados, (3) perguntas sobre prazo e orçamento devem vir depois de explorar o escopo — caso contrário o cliente ancora tudo em "quero barato e rápido", (4) um bom briefing já resolve 30% dos problemas que surgem no meio do projeto.

**Tipo de projeto:** [ex: site institucional / e-commerce / sistema web / app mobile / automação / API / integração]
**Setor do cliente:** [ex: saúde / educação / varejo / serviços / restaurante / imobiliário]
**Tamanho do cliente:** [ex: profissional solo / micro empresa / empresa média]
**Nível técnico do cliente:** [ex: leigo total / tem noção básica / é técnico também]
**Integrações que provavelmente serão necessárias:** [ex: pagamento, e-mail, WhatsApp, ERP, CRM]

Entregue:

**1. Perguntas de Contexto e Objetivo**
5-7 perguntas para entender o negócio do cliente, o problema que o projeto resolve e o que o sucesso significa para ele.

**2. Perguntas de Funcionalidades**
Perguntas específicas sobre as features do projeto, com opções de resposta para facilitar para clientes não técnicos.

**3. Perguntas de Design e Experiência**
Referências visuais, identidade, público-alvo, dispositivos e acessibilidade.

**4. Perguntas de Integrações e Infraestrutura**
Tecnologias existentes, plataformas que precisa integrar, hospedagem, domínio e manutenção futura.

**5. Perguntas de Prazo, Orçamento e Prioridades**
Como o cliente vê o prazo ideal, o que é prioridade para a primeira entrega e qual o investimento previsto.

**6. Campo de Materiais Necessários**
Lista do que você vai precisar que o cliente forneça: logo, textos, fotos, acessos, etc.
```

## Exemplo de uso

### Input
- Projeto: E-commerce
- Setor: Moda / Vestuário
- Cliente: Micro empresa, loja física que quer vender online
- Nível técnico: Leigo total
- Integrações: Pagamento, frete, Instagram
