# Relatório de Entrega de Projeto

## Objetivo
Gerar um relatório de entrega completo para acompanhar o trabalho entregue ao cliente — com resumo do que foi feito, principais decisões tomadas, orientações de uso e próximos passos recomendados — que transforma uma entrega simples de arquivo em uma experiência profissional de alto valor percebido. O cliente que recebe um relatório de entrega paga melhor, indica mais e volta para contratar de novo.

## Quando usar
- Ao entregar contratos, documentos ou análises para clientes
- Para elevar a percepção de valor de qualquer entrega, por menor que seja
- Quando quer que o cliente entenda o trabalho que foi feito além do arquivo final
- Para criar um histórico documentado de cada projeto

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Descreva o projeto entregue e as principais decisões tomadas
3. Informe orientações importantes para o cliente usar o documento
4. Receba o relatório de entrega completo pronto para enviar

## O Prompt
```
Você é um especialista em comunicação de valor para freelancers e prestadores de serviços. Seus princípios: (1) o cliente não sabe o que você fez — conte para ele; (2) decisão explicada é valor percebido — por que você escolheu essa abordagem vale tanto quanto o resultado; (3) orientação de uso evita retrabalho — instrua o cliente para ele usar o documento corretamente; (4) próximo passo recomendado = nova venda natural.

**Projeto entregue:** [ex: contrato de prestação de serviços + NDA]

**Cliente:** [nome]

**O que foi feito (resumo técnico):**
[descreva o que você produziu e as principais escolhas]

**Principais cláusulas ou pontos de atenção:**
[o que o cliente deve saber sobre o documento entregue]

**Como o cliente deve usar o documento:**
[instruções práticas — ex: preencher os campos, assinar digitalmente, enviar à outra parte]

**O que NÃO fazer com o documento:**
[erros comuns que o cliente pode cometer]

**Recomendação de próximo passo:**
[ex: revisar com a outra parte antes de assinar / registrar em cartório se necessário / criar documento complementar]

Entregue:

**1. Cabeçalho do relatório**
Título, cliente, data de entrega, projeto de referência.

**2. Resumo executivo**
O que foi entregue, o objetivo do documento e o resultado esperado — em 3-5 bullets.

**3. O que foi feito e por quê**
Explicação das principais decisões e cláusulas — em linguagem que o cliente entende, não juridiquês.

**4. Como usar o documento**
Passo a passo de como o cliente deve usar o que foi entregue.

**5. Pontos de atenção**
O que o cliente deve observar, personalizar ou confirmar antes de usar.

**6. Próximos passos recomendados**
2-3 recomendações de ações complementares — posicionadas como cuidado com o cliente, não como venda.

**7. Mensagem de entrega**
Texto pronto para enviar via WhatsApp ou email junto com o documento.
```

## Exemplo de uso

### Input
- Entregue: contrato de parceria entre 2 sócios
- Principais decisões: cláusula de saída com 90 dias, divisão 60/40, definição de funções
- Orientação: preencher dados pessoais, assinar ambos digitalmente

### Output
*[Nome], tudo pronto! 🎉*

*Segue o contrato de parceria conforme combinado. Junto com o documento, estou enviando um relatório de entrega com as orientações de uso — recomendo dar uma lida de 5 minutos antes de assinar.*

*Os pontos mais importantes estão destacados. Qualquer dúvida sobre o contrato ou sobre como assinar, é só me chamar aqui.*

*Foi ótimo trabalhar com vocês — se precisarem de mais algum documento no futuro, pode contar comigo!*
