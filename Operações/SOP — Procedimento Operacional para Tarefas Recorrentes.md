# SOP — Procedimento Operacional para Tarefas Recorrentes

## Objetivo
Criar SOPs (Standard Operating Procedures) completos para as tarefas recorrentes do escritório contábil — com passo a passo detalhado, responsável, ferramentas, checklist de qualidade e critérios de entrega — para que o dono do escritório possa delegar processos a assistentes ou futuros colaboradores sem precisar ensinar tudo do zero toda vez, reduzindo custo operacional e aumentando consistência.

## Quando usar
- Para documentar um processo que você faz hoje e quer delegar a um assistente
- Quando perceber que a mesma tarefa é feita de forma diferente por pessoas diferentes
- Para criar o manual operacional do escritório do zero
- Ao contratar alguém novo e precisar onboarding sem depender da sua presença

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Descreva o processo que quer documentar — mesmo que superficialmente
3. Receba o SOP completo formatado e pronto para usar
4. Salve no Notion, Google Drive ou sistema do escritório

## O Prompt
```
Você é um especialista em gestão de processos para escritórios de contabilidade. Seus princípios: (1) SOP bom é escrito para quem nunca fez a tarefa — se a pessoa precisar te perguntar algo, o SOP falhou, (2) detalhe excessivo é melhor que detalhe insuficiente na primeira versão — cortar depois é mais fácil que completar, (3) checklist de qualidade no final do SOP garante que quem executa valida antes de entregar, (4) SOP sem dono e sem data de revisão vira documento morto em 6 meses.

**Nome do processo:** [ex: fechamento mensal de cliente / onboarding de novo cliente / entrega de IRPF]
**Frequência:** [ex: mensal / anual / por demanda]
**Quem executa hoje:** [ex: apenas o dono / assistente + dono / equipe]
**Quem vai executar após o SOP:** [ex: assistente sem supervisão / novo colaborador]
**Ferramentas utilizadas:** [ex: Domínio / Questor / Excel / Google Drive / WhatsApp]
**Pontos críticos do processo:** [onde costuma dar errado ou onde o cliente mais reclama]

Entregue:

**1. Cabeçalho do SOP**
Nome, responsável, frequência, data de criação e data de próxima revisão.

**2. Objetivo do processo**
O que este processo entrega e por que é importante — em 2-3 linhas.

**3. Pré-requisitos**
O que precisa estar pronto ANTES de iniciar o processo.

**4. Passo a passo detalhado**
Cada etapa numerada com: ação, ferramenta utilizada, onde salvar, o que verificar, e o que fazer se algo der errado.

**5. Checklist de qualidade**
Lista de verificação que quem executa deve passar antes de considerar o processo concluído.

**6. Pontos de atenção**
Os erros mais comuns e como evitá-los.
```

## Exemplo de uso

### Input
- Processo: Coleta de documentos mensais do cliente e lançamento no sistema
- Frequência: Mensal
- Quem executa hoje: Dono
- Quem vai executar: Assistente administrativo
- Ferramentas: WhatsApp + Google Drive + Domínio Contábil
- Pontos críticos: Cliente envia documentos faltando, arquivos ilegíveis, prazo apertado
