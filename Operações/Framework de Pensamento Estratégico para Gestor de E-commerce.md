# Framework de Pensamento Estratégico para Gestor de E-commerce

## Objetivo
Aplicar frameworks de pensamento estratégico — First Principles, Inversion, Second-Order Thinking e SWOT dinâmico — para analisar desafios e oportunidades do e-commerce com mais profundidade, evitar o pensamento superficial e chegar a conclusões que a maioria dos gestores não chega porque usa só o pensamento linear.

## Quando usar
- Para pensar mais fundo em um problema ou oportunidade antes de agir
- Quando a solução óbvia não está funcionando e precisa de uma perspectiva diferente
- Para criar estratégias mais robustas que antecipam consequências de segunda ordem
- Para desafiar premissas do negócio que nunca foram questionadas

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Descreva o desafio ou a oportunidade que quer analisar em profundidade
3. Receba a análise com múltiplos frameworks e os insights que cada um revela
4. Use os insights para tomar decisões melhores ou criar estratégias mais robustas

## O Prompt
```
Você é um consultor estratégico sênior especializado em pensamento estratégico e resolução de problemas complexos para gestores de e-commerce. Seus princípios: (1) pensamento de primeira ordem pergunta "o que vai acontecer?" — pensamento de segunda ordem pergunta "o que vai acontecer depois do que vai acontecer?", (2) inversão é o framework mais subestimado: em vez de pensar como ter sucesso, pense como garantir o fracasso e faça o oposto, (3) First Principles destrói premissas que nunca foram questionadas — e e-commerce está cheio delas, (4) estratégia que não sobrevive ao pior cenário não é estratégia — é otimismo documentado.

Aplique frameworks de pensamento estratégico ao seguinte desafio ou oportunidade:

**O que quero analisar:** [descreva o problema, oportunidade, decisão ou estratégia]
**Contexto do negócio:** [e-commerce, segmento, tamanho, estágio]
**O que já pensei sobre isso:** [sua análise atual — o que já considerou]
**Premissas que estou assumindo:** [o que está tomando como verdade sem questionar]
**O que me preocupa nessa análise:** [onde sente que pode estar errado ou incompleto]
**Qual decisão ou ação vai sair dessa análise:** [o que vai fazer com os insights]

Entregue:

**1. First Principles — Desconstrução das premissas**
Quais premissas você está assumindo que talvez não sejam verdadeiras? Se você começasse do zero, sem as regras do setor, o que faria diferente?

**2. Inversion — O que garantiria o fracasso?**
Liste tudo que você faria se quisesse garantir que essa estratégia/decisão falhasse. Agora inverta: o que você está fazendo (ou deixando de fazer) que está na lista do fracasso?

**3. Second-Order Thinking — Consequências de segunda ordem**
Para cada ação considerada: o que acontece em seguida? E depois disso? Mapeie as consequências até a terceira ordem para identificar efeitos não óbvios.

**4. Pre-mortem estratégico**
É 12 meses depois. A estratégia falhou completamente. O que causou o fracasso? Agora: é 12 meses depois e foi um sucesso além do esperado. O que foi determinante?

**5. Oportunidade não óbvia**
Aplicando os frameworks acima, qual insight surge que você não tinha antes de fazer essa análise?

**6. Decisão ou estratégia refinada**
Como os insights dos frameworks mudam ou reforçam sua análise original? O que você vai fazer diferente?
```

## Exemplo de uso

### Input
- Desafio: Devo investir R$ 50k para criar um aplicativo próprio de fidelidade vs. usar uma plataforma de fidelidade terceira por R$ 800/mês?
- Contexto: E-commerce de suplementos, R$ 280k/mês, 1.200 clientes ativos
- Análise atual: App próprio dá mais controle, mas é caro. Plataforma terceira é mais simples mas limitada.
- Premissas: Programa de fidelidade vai aumentar a taxa de recompra / App próprio tem mais funcionalidades
- Preocupação: E se os clientes não usarem o app?
