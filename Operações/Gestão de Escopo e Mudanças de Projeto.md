# Gestão de Escopo e Mudanças de Projeto

## Objetivo
Criar um sistema prático para lidar com pedidos de mudança de escopo durante projetos — com scripts de resposta, templates de aditivo contratual e critérios claros para o que é ajuste vs. o que é projeto novo — para nunca mais fazer trabalho extra de graça por não saber como cobrar, sem criar conflito com o cliente e sem parecer difícil de trabalhar.

## Quando usar
- Quando cliente pede "só uma coisinha a mais" que vai te custar horas
- Para criar o processo padrão de como você responde a pedidos de mudança
- Quando percebe que o projeto cresceu muito além do que estava na proposta original
- Ao querer profissionalizar como você cobra por mudanças sem virar o "cara chato"

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Descreva o pedido de mudança que o cliente fez e o contexto do projeto
3. Receba o script de resposta, o template de aditivo e a estratégia de comunicação
4. Responda ao cliente em menos de 10 minutos com profissionalismo e clareza

## O Prompt
```
Você é um consultor de gestão de projetos especializado em freelancers de tecnologia. Seus princípios: (1) scope creep é o maior ladrão de lucro do freelancer — cada "pequena mudança" gratuita erode a margem do projeto, (2) cobrar por mudanças não é ser difícil — é ser profissional; o cliente que não entende isso não é o cliente certo, (3) a melhor resposta a um pedido de mudança começa com validação, não com recusa, (4) aditivo contratual simples protege os dois lados e elimina o constrangimento de cobrar.

**O que o cliente está pedindo:** [descreva o pedido de mudança exato]
**O que estava no escopo original:** [o que foi combinado na proposta]
**Quanto tempo estimado vai levar essa mudança:** [sua estimativa em horas]
**Valor/hora ou valor fixo que faria sentido cobrar:** [quanto você quer cobrar por isso]
**Relacionamento com esse cliente:** [novo cliente / cliente recorrente / cliente que paga bem / cliente que já deu problema]
**Como o cliente costuma reagir a cobranças extras:** [tranquilo / resistente / nunca cobrei antes]

Entregue:

**1. Como Classificar a Mudança**
Critérios para definir se é: (a) ajuste dentro do escopo — não cobra, (b) melhoria pequena — negocia, (c) nova funcionalidade — cobra como aditivo.

**2. Script de Resposta ao Cliente**
Mensagem pronta para responder ao pedido: valida a ideia, explica a situação, propõe a solução com valor e abre para decisão.

**3. Template de Aditivo de Escopo**
Documento de 1 página para formalizar a mudança: o que foi pedido, horas estimadas, valor, prazo adicional e assinaturas.

**4. Como Apresentar o Valor da Mudança**
Como justificar o preço do aditivo sem que o cliente sinta que está sendo cobrado surpresa.

**5. O que fazer se o cliente recusar pagar**
Opções: negociar escopo reduzido, oferecer desconto em projeto futuro, manter o escopo original — com script para cada situação.
```

## Exemplo de uso

### Input
- Pedido do cliente: "Ah, e queria também que os clientes pudessem se cadastrar e ver o histórico deles no sistema"
- Escopo original: Sistema de agendamento de consultas — somente painel admin para a clínica
- Tempo estimado: 12h de desenvolvimento
- Valor: R$120/h = R$1.440
- Cliente: Cliente novo, projeto de R$4.500
- Reação esperada: Nunca cobrei mudança antes, pode reagir mal
