# Checklist de Entrega de Projeto Dev

## Objetivo
Criar um checklist completo e personalizado para a entrega de projetos de desenvolvimento — cobrindo qualidade de código, testes, documentação, deploy, comunicação com cliente e encerramento formal — para nunca mais entregar um projeto esquecendo algo importante, causar boa impressão no momento final e proteger você de chamados pós-entrega por coisas que poderiam ter sido verificadas antes.

## Quando usar
- Antes de qualquer entrega de projeto para cliente
- Para criar um processo padrão de entrega que se aplica a todos os seus projetos
- Quando você entrega e o cliente sempre encontra algo que você esqueceu
- Para ter um ritual de encerramento que formaliza o fim do projeto e abre espaço para upsell

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Descreva o tipo de projeto, as tecnologias usadas e o que você vai entregar
3. Receba o checklist completo dividido por categoria com itens prontos para marcar
4. Salve no Notion ou como template e use em todo projeto antes de dizer "tá pronto"

## O Prompt
```
Você é um desenvolvedor sênior com foco em qualidade de entrega e experiência do cliente. Seus princípios: (1) o cliente não avalia o código — avalia a experiência de receber o projeto e usar o resultado, (2) entrega sem documentação é entrega pela metade — o cliente vai te chamar mil vezes depois, (3) checklist de qualidade técnica e checklist de comunicação com cliente são diferentes e precisam acontecer juntos, (4) o momento da entrega é o melhor momento para plantar o próximo projeto — cliente satisfeito na entrega é cliente que indica e volta.

**Tipo de projeto:** [ex: site / e-commerce / sistema web / app / API / automação]
**Tecnologias usadas:** [ex: React, Node.js, PostgreSQL / WordPress / Python + AWS Lambda]
**O que vai ser entregue:** [descreva os entregáveis — ex: código-fonte, link do sistema, documentação, credenciais]
**O cliente vai mexer no sistema sozinho?** [sim — é leigo / sim — é técnico / não — só usa]
**Onde está hospedado:** [ex: Vercel, AWS, servidor do cliente, Hostinger, local]

Entregue:

**1. Checklist Técnico (pré-entrega)**
Verificações de qualidade de código, testes, performance, segurança e deploy.

**2. Checklist de Documentação**
O que documentar antes de entregar: README, manual do usuário, credenciais, arquitetura e decisões técnicas.

**3. Checklist de Comunicação com Cliente**
Como comunicar a entrega: o que incluir no e-mail/mensagem de entrega, como apresentar o projeto, o que pedir de confirmação.

**4. Checklist Pós-Entrega (primeiras 48h)**
O que verificar depois que o cliente começou a usar: erros inesperados, dúvidas comuns, estabilidade do sistema.

**5. Template de E-mail de Entrega**
Mensagem profissional de entrega do projeto com o que incluir para impressionar e facilitar a transição.

**6. Abertura para Próximo Projeto (Upsell)**
Como mencionar na entrega os próximos passos naturais sem parecer que está empurrando venda.
```

## Exemplo de uso

### Input
- Projeto: Landing page de captação de leads
- Tecnologias: HTML/CSS/JS + integração Mailchimp + formulário
- Entregáveis: Link da página ao vivo, código-fonte no GitHub, acesso ao painel Mailchimp
- Cliente: Leigo, vai querer editar textos sozinho
- Hospedagem: Netlify (domínio do cliente apontado)
