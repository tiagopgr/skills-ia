# SOP de Processos Internos do Escritório de Advocacia

## Objetivo
Criar procedimentos operacionais padrão (SOPs) para o escritório de advocacia — documentando passo a passo como cada processo deve ser executado, quem faz o quê e quando — para que o advogado pare de depender só da própria memória, reduza erros operacionais e consiga delegar com mais segurança quando tiver equipe.

## Quando usar
- Quando quiser documentar como funciona a entrada de um novo cliente no escritório
- Quando precisar padronizar o processo de elaboração e envio de petições
- Quando quiser criar um checklist de onboarding de cliente novo
- Quando quiser organizar como funciona o controle de prazos processuais
- Quando for contratar um assistente e precisar treinar sem depender da sua presença

## Como usar
1. Copie o prompt abaixo no Claude ou ChatGPT
2. Descreva o processo que quer documentar e como ele funciona hoje
3. Receba o SOP estruturado, com checklist e pontos de controle
4. Revise, ajuste à sua realidade e salve como documento de referência

## O Prompt
```
Você é um consultor de operações especializado em escritórios de advocacia e negócios jurídicos. Seus princípios: (1) um SOP bem feito deve ser executável por alguém que nunca fez antes — clareza acima de sofisticação, (2) todo processo precisa de um "dono" responsável — mesmo que seja só uma pessoa fazendo tudo, (3) pontos de controle evitam que erros cheguem ao cliente — defina checkpoints, não só passos, (4) um SOP que ninguém lê não serve — formato prático e visual é parte do trabalho.

Quero documentar o seguinte processo do meu escritório:

**Nome do processo:** [ex: Entrada de novo cliente / Elaboração de petição / Controle de prazos / Envio de proposta / Fechamento de contrato]
**Como funciona hoje (descreva livremente):** [conte como você faz atualmente — mesmo que seja desorganizado]
**Quem executa:** [só você / você + assistente / equipe]
**Ferramentas que usa:** [ex: e-mail, WhatsApp, planilha, PJe, software jurídico]
**Qual a maior dor neste processo hoje:** [onde dá mais erro, esquece, atrasa]
**Resultado esperado quando o processo termina:** [ex: cliente cadastrado + contrato assinado + pasta criada]

Entregue:

**1. SOP completo do processo**
Com as seções:
- Nome e objetivo do processo
- Gatilho (o que inicia o processo)
- Responsável(eis)
- Passo a passo numerado e detalhado
- Ferramentas necessárias por etapa
- Pontos de controle (onde checar antes de avançar)
- O que fazer se algo der errado
- Resultado esperado ao final

**2. Checklist resumida**
Versão enxuta do SOP em formato de checkbox para uso no dia a dia (pode ser colada num Notion, Google Docs ou impresa).

**3. Sugestões de melhoria**
3 ajustes no processo que aumentariam eficiência ou reduziriam risco de erro.
```

## Exemplo de uso

### Input
- Processo: Entrada de novo cliente
- Como funciona: Cliente liga, a gente conversa, mando proposta por WhatsApp, se aceitar assino contrato por e-mail
- Executa: Só eu
- Ferramentas: WhatsApp, e-mail, Word para contrato
- Maior dor: Às vezes esqueço de pegar todos os documentos e fico pedindo depois
- Resultado esperado: Cliente com contrato assinado + documentos recebidos + pasta criada

### Output
"☐ 1. Realizar reunião/call de diagnóstico (máx 30 min)
☐ 2. Preencher ficha de cadastro do cliente (nome, CPF/CNPJ, contato, problema)
☐ 3. Enviar proposta de honorários em até 24h
☐ 4. Aguardar aceite — follow-up em 48h se não houver resposta
☐ 5. Enviar contrato para assinatura (DocuSign ou e-mail)
☐ 6. Solicitar documentos necessários (lista padrão por tipo de caso)
☐ 7. Criar pasta do cliente (física e/ou digital)
☐ 8. Registrar no controle de clientes ativos..."
